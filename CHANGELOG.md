# Changelog

## v1.6.37

- 修复启动器图标被误接回旧矢量 mark 的问题
- 启动器普通图标和圆形图标的 adaptive icon 前景重新改为图片版 `ic_launcher_art`
- 删除旧的 `ic_payki_mark` / `ic_payki_mark_monochrome` / `ic_launcher_foreground` / `ic_launcher_monochrome` 资源，避免后续再次误用
- 版本号升级到 `1.6.37` / `versionCode 109`

## v1.6.36

- 待办提醒、日程提醒、待办批量、日程批量和自定义延后输入框旁新增圆形问号帮助按钮
- 新增共享输入语法帮助弹窗，说明提醒时间、待办批量、日程批量和延后时间的填写规则
- 更新内置 Wiki 到 1.6.36，补充统一提醒时间写法、待办批量逗号语法、日程批量 `Remind=` 和自定义延后说明
- 无障碍提醒浮层的自定义延后行增加问号提示按钮
- 版本号升级到 `1.6.36` / `versionCode 108`

## v1.6.35

- 待办批量添加改为轻量逗号语法：`DDL时间,任务名称,提醒时间`，默认使用默认分组、响铃和震动设置
- 待办批量添加的提醒时间只需填写一个提醒点，支持 `5`、`16:30`、`05-10 15:00`、`2026-05-10 14:30`
- 日程批量导入的 `Remind=` 字段接入同一套提醒时间识别逻辑，不再只局限于 `15m / 1h / 2d`
- 全屏提醒页和无障碍提醒浮层的“自定义延后”输入支持分钟数或具体未来时刻
- 版本号升级到 `1.6.35` / `versionCode 107`

## v1.6.34

- 待办与日程提醒时间统一改为文本输入：支持 `5,15,16:30,05-10 15:00,2026-05-10 14:30` 这种逗号分隔多提醒写法
- 提醒输入会校验每一项：晚于 DDL / 日程开始时间、格式非法或新建时已过期都会标红，并禁用保存按钮
- 普通待办正式接入 `reminderOffsetsCsv` 多提醒链路，避免待办只能保存和派发单个提醒点
- 新增待办批量导入弹窗，支持按行导入 DDL、标题、备注、分组、响铃震动和多提醒配置
- 每日看板 / 我的任务增加显性的“批量添加待办 / 批量添加日程”按钮，日历页顶部也新增独立“批量”按钮
- 版本号升级到 `1.6.34` / `versionCode 106`

## v1.6.33

- 启动器 adaptive icon 前景重新接回安全区矢量 PaykiTodo mark，避免误回退到旧位图图标链路
