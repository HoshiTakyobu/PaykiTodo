# PaykiTodo 后续功能预测与路线建议

## 说明

- 这不是强制承诺清单，而是根据当前代码结构、你反复强调的痛点、以及你的使用习惯推导出的高概率下一阶段方向。
- 推断依据主要来自你长期关注的几点：强提醒、锁屏可靠性、课程/值班类日程、重复习惯、自律场景、本地可控、阅读与输入效率、以及与 Obsidian / 坚果云式工作流的兼容诉求。

## 一、高概率继续开发的方向

### 1. 更强的提醒可靠性专项

- 针对不同 ROM 的锁屏、后台、解锁回到提醒页行为继续打补丁。
- 补充更细的提醒自检机制，例如最近一次提醒派发链路的状态记录。
- 在设置页中加入“提醒链路测试”入口，减少每次都要手动等一分钟的烦躁感。

### 2. 更适合课程表与值班表的日历增强

- 课程表模板化导入。
- 学期级循环日程生成器。
- 固定周模板复制，例如“一键导入本周值班安排”。
- 更完整的日历视图，例如周视图、月视图、列表视图切换。

### 3. 更强的批量导入能力

- 语法继续扩展，例如支持颜色、提醒方式、全天标记、分组映射。
- 支持从剪贴板直接粘贴并识别。
- 支持 CSV / TSV / 课表文本 / ics 等更多来源格式。

### 4. 循环任务与循环日程的高级编辑

- 单次例外规则，例如“本周暂停一次，但不破坏整个循环系列”。
- 系列拆分工具，例如“从这一周开始改为新时间”。
- 更直观的系列级预览界面，方便检查生成结果。

### 5. 数据同步与外部工作流打通

- 备份目录与坚果云目录更顺滑联动。
- 更明确的 Obsidian 兼容导出格式。
- 可能出现“定时自动导出 Markdown / JSON 快照”的能力。
- 如果未来继续推进，很可能会出现轻量级“伪云同步”，但仍以本地优先为主。

## 二、基于你的使用习惯，极可能受欢迎的功能

### 1. 自律强化场景功能

- 药物提醒、睡眠提醒、课程出门提醒、复盘提醒等模板化创建。
- 错过后自动升级提醒强度，例如从普通通知升级到全屏提醒。
- 连续错过任务的集中处理面板。

### 2. 快速录入能力继续加强

- 更适合键盘输入的“快速添加模式”。
- 一次性输入多条待办或多条日程。
- 常用文本自动补全，例如地点、课程名前缀、固定提醒模式。

### 3. 信息密度更高但更整洁的界面

- 首页支持更多视图模式，例如紧凑模式 / 卡片模式。
- 历史记录加入筛选与搜索。
- 日历中支持更清晰的颜色图例与分组过滤。

### 4. 更贴近无障碍与高可读性的交互

- 更大的触控目标。
- 更明确的状态朗读或无障碍提示文案。
- 更简洁、更少跳转层级的提醒处理路径。

## 三、从当前代码结构推断，技术上顺势可做的功能

### 1. 统计与复盘

- 每日完成率。
- 每周错过率。
- 按分组统计任务完成情况。
- 日程/待办的活跃度趋势图。

### 2. 数据导出增强

- 导出为 Markdown。
- 导出为更适合 Obsidian 的目录结构。
- 导出时附带分组、完成状态、错过状态、提醒设置。

### 3. 日历与待办联动

- 把待办一键转成日程。
- 把日程一键拆成待办清单。
- 从日历空闲时段自动建议安排任务。

### 4. 自定义提醒资源

- 提供多套内置提醒音。
- 针对不同分组设置不同铃声或提醒策略。
- 按待办/日程类型分别设置默认提醒方式。

## 四、我认为最值得优先做的下一阶段路线

### 第一优先级

- 锁屏/后台提醒稳定性继续专项打磨。
- 一键提醒链路测试功能。
- 日历与批量导入继续优化。

### 第二优先级

- 备份与 Obsidian / 坚果云工作流衔接。
- 循环任务的高级编辑与例外规则。
- 历史记录和错过任务的检索与复盘。

### 第三优先级

- 周视图 / 月视图等更完整日历界面。
- 统计面板。
- 更丰富的内置提醒音与视觉主题。

## 五、目前不建议过早投入的方向

- 复杂云账号体系。
- 多人协作。
- 依赖大型后端服务的在线同步。
- 与当前“本地强提醒”主线关系较弱的社交化功能。

## 六、结论

- 这个项目当前最强的核心竞争力不是“普通待办”，而是“本地单机 + 强提醒 + 日历/待办一体化 + 高可定制录入”。
- 你最在意的也一直不是花哨功能，而是：真的能提醒到、真的适合自己、真的足够快。
- 因此后续最可能继续长出来的功能，不会偏向“社交”和“云协作”，而会继续偏向：更强提醒、更快录入、更稳定锁屏、更适合课程/值班/自律场景，以及与本地知识管理工作流更顺滑地衔接。
