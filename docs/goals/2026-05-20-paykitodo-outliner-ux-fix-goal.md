## 背景

v1.12.0-1.12.9 已经实现了规划台 Outliner 的数据层和基本 UI，但**输入体验严重偏离用户预期**。

用户的反馈：
- "我希望在规划台我是和写笔记一样，那种整个区域全是敲键盘的那种"
- "现在反倒得我一个一个事项敲了"
- "我希望写了一行之后，按了手机键盘的回车，就变成有圆圈、有向下展开的那个标志了"
- "新手教程好像一点没改，我想跟着那个教程走的，结果发现教程和现在版本根本不匹配"
- "三个点里面的功能太多了，我根本不会操作"

**核心问题**：当前 Outliner 是"结构化列表 + 逐条添加"的体验，但用户要的是"像备忘录一样打字，回车后自动变成节点"的体验。换了个形式，本质上还是在填表。

## 理想体验描述

### 编辑模式（默认）

1. 打开规划台 → 一整片空白文本区域，光标闪烁，像备忘录/笔记 app
2. 直接打字 "入党资料要写完"，按手机键盘回车
3. 这一行自动变成一个节点：左边出现空心圆圈（待办标记）+ 折叠箭头
4. 光标自动跳到下一行，继续打字 "数据库复习"，回车 → 又一个节点
5. 整个过程就像在备忘录里一行一行写东西，只不过每一行写完后会自动获得「待办」的身份
6. 编辑模式下保持干净，每行右边**不显示**额外按钮

### 字段识别（在编辑模式内直接写）

用户可以在一行文本里按固定顺序写字段：

```
时间, 事件名, 地点
```

识别规则：
- 纯靠**顺序**和**分隔符**（英文逗号、分号、或空格）判断
- **不靠任何特殊符号**（不靠 `@`、不靠 `#`、不靠任何前缀）
- 第一个字段如果能解析为时间 → 识别为时间
- 最后一个字段如果前面已有时间和事件名 → 识别为地点
- 没有时间 → 整行就是事件名，无 DDL 待办

示例：
- `15:00-17:00, 写论文, 图书馆3楼` → 时间段=15:00-17:00，事件=写论文，地点=图书馆3楼 → 日程
- `明天下午3点, 交论文` → 时间点=明天下午3点，事件=交论文 → DDL 待办
- `整理保研材料` → 纯文字 → 无 DDL 待办
- `15:00-17:00 写论文 图书馆3楼` → 空格分隔，同样识别

### 展开箭头 + 子任务

6. 箭头的作用是「展开/折叠子任务列表」，不是「进入编辑这条任务」
7. 点箭头 → 展开这条下面的子任务区域
8. 子任务区域也是同样的输入体验：打字 → 回车 → 变成子节点
9. 子任务可以无限套娃

### 父子关系与待办身份

10. **一旦某个节点有了子项，这个父节点就不再是独立的待办/日程** — 它变成"项目/分类标题"，具体的待办/日程身份落在子项上
11. 父节点不创建 linkedTodoId，不出现在待办页面
12. 只有叶子节点（没有子项的节点）才是真正的待办/日程

### 完成状态

13. 子项完成 → 圆圈勾选 + 文字颜色变深（灰→黑）+ 删除线
14. **所有子项都完成 → 父项自动变成完成状态**（同样的视觉效果）
15. 双向同步不变：待办页面勾选 → 规划台同步显示；规划台勾选 → 待办页面同步

### 预览模式

16. 有一个"编辑/预览"切换（类似现在的 Markdown 编辑/预览切换）
17. **预览模式下**：文本不可编辑，每行右边出现三个点按钮
18. 点三个点 → 弹出完整字段编辑面板（和现有待办/日程编辑器一样：提醒、分组、重复、备注、倒数日等）
19. 预览模式用于查看和精细配置，编辑模式用于快速输入

## 具体改动

### U1. 输入体验重写

#### 当前问题
`PlanningOutlineEditor` 是一个 `LazyColumn`，每个 node 是独立的 `OutlinedTextField` 卡片。用户必须点"新增"或在已有节点里按 Enter 才能创建新节点。

#### 目标
改为一个**连续的文本输入区域**，视觉上像备忘录。每按一次回车，上面那一行就"固化"为一个节点（出现圆圈和箭头），光标留在新的空行继续输入。

#### 实现思路

方案：**混合模式** — 已有节点渲染为带圆圈/箭头的行，底部始终有一个活跃的输入行

```
┌─────────────────────────────────────┐
│ ○ ▶ 入党资料要写完                   │  ← 已固化的节点
│ ○ ▶ 数据库复习                       │  ← 已固化的节点
│ ○ ▶ 15:00-17:00, 写论文, 图书馆3楼   │  ← 已固化的节点
│                                       │
│ |                                     │  ← 活跃输入行（光标在这里）
└─────────────────────────────────────┘
```

- 用户在活跃输入行打字 → 按回车 → 文本被创建为新 node → 输入行清空，光标留在原地
- 已固化的节点：点击可以进入编辑（文本变为可编辑状态），失焦后保存
- 整体视觉：不要卡片边框，不要每行一个 Surface，保持像纯文本列表的感觉

#### 子任务输入

- 点击某个节点的展开箭头 → 展开子任务区域
- 子任务区域底部同样有一个活跃输入行（缩进显示）
- 在子任务输入行打字 → 回车 → 创建为该节点的子节点
- 子任务区域可以继续展开下一层（套娃）

#### 键盘交互保留

- Enter：创建新节点（在活跃输入行）或创建 sibling（在已有节点编辑时）
- Tab：缩进（变成上一个节点的子项）
- Shift+Tab：取消缩进
- Backspace（空行）：删除当前节点

### U2. 父子关系逻辑修正

#### 当前问题
每个 node 创建时都自动创建 linkedTodoId，包括有子项的父节点。

#### 目标
- 叶子节点（无子项）→ 自动创建 linkedTodoId，是真正的待办/日程
- 有子项的节点 → **不创建** linkedTodoId，只是组织标题
- 当用户给一个已有 linkedTodoId 的节点添加第一个子项时 → 自动解除该节点的 linkedTodoId（删除对应的 TodoItem）
- 当用户删除一个节点的所有子项时 → 该节点重新变为叶子节点 → 自动创建 linkedTodoId

#### 完成状态联动
- 所有子项 completed = true → 父项自动 completed = true
- 任一子项 completed = false → 父项 completed = false
- 这个联动是自动计算的，不需要用户手动操作

### U3. 预览模式

#### 当前问题
没有独立的预览模式。编辑和配置混在一起。

#### 目标

工具栏加一个"编辑/预览"切换按钮：

**编辑模式**（默认）：
- 文本可编辑
- 每行只显示：缩进 + 圆圈 + 箭头 + 文本 + 时间/地点摘要
- 底部有活跃输入行
- 干净、专注于输入

**预览模式**：
- 文本不可编辑
- 每行右边出现 `⋯` 三点按钮
- 点击三点 → 弹出 BottomSheet，内容和现有待办/日程编辑器一致：
  - 提醒设置
  - 分组选择
  - 重复/循环
  - 备注
  - 倒数日开关
  - 打卡追踪开关（日程）
  - 删除
- 预览模式下也能点圆圈完成/取消完成

### U4. 新手教程更新

#### 当前问题
教程还在讲旧的 Markdown 编辑器工作流，和 v1.12.x 的 Outliner 完全不匹配。

#### 目标

重写规划台使用说明（`helpSheetVisible` 弹出的内容），内容覆盖：

1. **基本输入**：打字 → 回车 → 变成待办节点
2. **带时间的写法**：`时间, 事件名, 地点`（逗号/分号/空格分隔，按顺序识别）
3. **子任务**：点箭头展开 → 在下面继续写
4. **完成**：点圆圈勾选，或在待办页面勾选（双向同步）
5. **补充字段**：切换到预览模式 → 点三个点 → 设置提醒/分组/重复等
6. **Markdown 兼容**：仍可切换到 Markdown 模式导入/导出

教程用 2-3 页卡片式引导，每页一个核心概念，配简单示例。

### U5. 溢出菜单精简

#### 当前问题
三个点菜单里功能太多，用户不知道怎么操作。

#### 目标

精简为：
- 新建文档
- 重命名
- 使用说明
- 归档
- 删除

移除或合并到其他入口：
- "从图片识别日程" → 移到快速捕获入口（长按图标"拍照添加"已经覆盖了这个场景）
- Markdown 导入/导出 → 放在 Markdown 兼容模式的工具栏里，不在主菜单

### U6. 完成状态视觉修正

#### 当前问题
完成状态的颜色用的是 Material 主题色，不够明显。

#### 目标（严格按用户描述）

- **未完成**：文字颜色偏灰（`onSurfaceVariant`），圆圈空心
- **已完成**：文字颜色变深/变黑（`onSurface`）+ 删除线，圆圈实心勾选
- 像 Obsidian 渲染 `- [x]` 的效果
