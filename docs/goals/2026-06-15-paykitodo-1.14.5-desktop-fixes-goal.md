### 任务 1：日历连续滚动恢复 ⭐⭐⭐⭐⭐
**优先级**：高（架构性改动，用户强烈需求）

**问题描述**：
日历界面左右滑动变成了一页一页的分页滚动，之前版本是连续平滑滚动。用户明确要求恢复连续滚动体验。

**根本原因**：
版本 1.13.10 左右将 `horizontalScroll` 改为了 `HorizontalPager`，导致滚动行为从连续变为分页。

**技术细节**：
- **文件**：`app/src/main/java/com/example/todoalarm/ui/CalendarPanel.kt`
- **位置**：L515 附近的 `HorizontalPager` 组件
- **历史参考**：版本 1.3.3 使用的是 `horizontalScroll` 修饰符

**实现要求**：
1. 将 `HorizontalPager` 改回使用 `horizontalScroll` 修饰符
2. 保持日历列的布局逻辑（每天一列）
3. 恢复平滑的连续滚动体验
4. 确保滚动到今天的定位逻辑仍然正常工作
5. 测试滚动流畅性和边界情况

**注意事项**：
- 这是架构性变更，可能需要重构日期列的渲染方式
- 原 `HorizontalPager` 的页面状态管理需要改为滚动偏移管理
- 确保性能不受影响（大量日期列的渲染）

### 任务 3：日程颜色显示不一致 ⭐⭐⭐
**优先级**：高（逻辑错误）

**问题描述**：
在桌面端编辑日程时选择蓝色，但在日程时间轴界面上显示的却是绿色（分组的默认颜色）。用户选择的自定义颜色没有生效。

**根本原因**：
渲染日程卡片时，分组颜色优先级高于自定义颜色，导致自定义颜色被覆盖。

**技术细节**：
- **文件**：`app/src/main/assets/desktop-web/app.js`
- **涉及函数**：日程卡片渲染函数（可能是 `renderEventCard` 或类似名称）

**实现要求**：
1. 定位日程时间轴中日程卡片的渲染逻辑
2. 检查颜色获取的优先级顺序：
   ```javascript
   // 错误的优先级（当前）
   const color = item.groupColor || item.accentColorHex;

   // 正确的优先级（应改为）
   const color = item.accentColorHex || item.groupColor;
   ```
3. 确保自定义颜色（`accentColorHex`）优先于分组颜色（`groupColor`）
4. 测试：
   - 有自定义颜色时，显示自定义颜色
   - 无自定义颜色时，显示分组颜色
   - 都没有时，显示默认颜色

**验证方法**：
创建一个测试日程，设置蓝色自定义颜色，确认时间轴上显示蓝色而非分组绿色。

### 任务 5：左侧边栏文字对比度提升 ⭐
**优先级**：中（可读性）

**问题描述**：
桌面端左侧边栏的文字几乎看不清，对比度不足。参考截图中红框标注区域的文字模糊不清。

**历史背景**：
版本 1.14.4 已将侧边栏背景色从浅色 `#f8fafc` 改回深蓝色 `#243246`，理论上深色背景应该用浅色文字，但当前文字对比度仍不足。

**技术细节**：
- **文件**：`app/src/main/assets/desktop-web/app.css`
- **涉及区域**：`.sidebar` 及其子元素的文字样式

**实现要求**：
1. 检查当前侧边栏文字颜色，确保与深蓝背景 `#243246` 有足够对比度
2. 推荐文字颜色：
   ```css
   .sidebar {
     background: #243246;
     color: #f8fafc; /* 浅色文字 */
   }

   .sidebar a,
   .sidebar button,
   .sidebar .menu-item {
     color: #f8fafc;
     font-weight: 500; /* 增加一点粗细 */
   }

   .sidebar .menu-item:hover {
     color: #ffffff;
     background: rgba(255, 255, 255, 0.1);
   }
   ```
3. 适当增加文字 `font-weight`（400 → 500）
4. 如果需要，可以略微增大 `font-size`（但不要超过 14px）
5. 确保选中状态、悬停状态的文字也清晰可见

**可访问性要求**：
文字与背景的对比度至少达到 WCAG AA 标准（对比度 ≥ 4.5:1）。
