# Changelog

## v1.9.18

- 电脑端待办管理列表新增分页和搜索：`/api/todos` 支持 `offset`、`limit` 和 `q`，浏览器端可搜索标题 / 备注 / 地点，并显示“已加载 / 总匹配”与“加载更多”
- 电脑端待办分页保持看板语义：已错过、今日 / 无 DDL、计划中、历史记录按业务优先级排序，避免无 DDL 活动待办被未来或历史数据挤到后面
- Android 今日看板小组件继续移除专注内容：运行时删除 Focus 行类型和专注卡分支，添加页静态预览也不再显示“今日已专注 / 自由专注”
- 小组件今日待办不再固定截断为 6 条，改由整体 RemoteViews 行数上限控制，较大尺寸的小组件可以露出更多待办
- 版本号升级到 `1.9.18` / `versionCode 212`

## v1.9.17

- 电脑端 Web 新增 `/api/todos` 和 `/api/events?start=...&end=...` 窄接口，待办管理和日程时间轴不再共用一个完整 `/api/snapshot`
- 电脑端首次连接仍只读取轻量每日看板；用户点击“加载完整待办列表”时只拉待办，切到日程时间轴时只拉当前可见日期范围内的活动日程
- 电脑端待办 / 日程保存、删除、完成、取消和规划台导入后的刷新改为按当前页面补齐数据，避免每次操作都重新读取全部待办和日程
- 无 DDL 活动待办在电脑端完整待办列表中继续归入 `今日待办`，与手机每日看板 / 我的任务语义保持一致
- 版本号升级到 `1.9.17` / `versionCode 211`

## v1.9.16

- 规划文档新增持久化公告提示字段 `hasAnnouncementHint`，保存正文、备份恢复和电脑端规划台同步时都会按 Markdown 正文重新计算，避免公告增删后看板读取旧状态
- 数据库版本升级到 `15`，`MIGRATION_14_15` 会为旧规划文档回填公告提示字段并创建 `archived + hasAnnouncementHint` 查询索引
- 每日看板、Android 小组件和电脑端轻量快照的公告候选查询改为走布尔字段和索引，不再对完整 Markdown 正文做 `LIKE` 扫描
- 补充 `PlanningNote` 默认公告提示单测，确认公告正文会自动标记，普通规划正文不会误标记
- 版本号升级到 `1.9.16` / `versionCode 210`

## v1.9.15

- 普通每日看板 / 我的任务状态不再携带完整规划台文档列表；规划台完整文档改为只有进入 `规划台` 页面时才单独订阅，避免长规划文档拖慢普通页面重组
- 每日看板公告改用规划文档公告提示窄查询：只读取疑似包含 `#公告`、`[!公告]`、`[!announcement]` 或 `公告:` 的未归档规划文档，再交给现有公告解析器做日期和正文过滤
- Android 今日看板小组件和电脑端 `/api/snapshot` 同步改用公告提示查询生成公告，不再为了看板首屏读取全部规划台文档
- 补充公告提示识别单测，确认常见公告入口能命中、普通规划正文不会误命中
- 版本号升级到 `1.9.15` / `versionCode 209`

## v1.9.14

- 手机端主状态继续瘦身：每日看板 / 我的任务不再订阅历史待办、完整日历模板和提醒链路诊断日志，历史记录、日历模板和设置诊断改为进入对应页面时才单独订阅
- 主界面待办数据改为按活动待办 Flow 查询，保留无 DDL 活动待办每天归入 `今日待办` 的语义，同时避免普通看板被历史记录列表拖慢
- 日历页打开时再订阅完整活动日程和课程模板；每日看板只订阅今明两天相关日程，减少远期日程对首页重组的影响
- 提醒链路诊断日志只在设置页订阅，不再随每日看板 / 我的任务状态更新
- 规划台本地规则识别移到后台线程执行；AI 失败回退本地规则时也不会占用 Compose 主线程解析长文档
- `AI 报告` 归档页改为按筛选和数量分页订阅，默认只加载最近 30 条，通知深链可按 ID 单独打开未在当前页内的旧报告
- 版本号升级到 `1.9.14` / `versionCode 208`

## v1.9.13

- 电脑端每日看板首屏改用轻量快照：连接后先请求 `/api/snapshot?scope=board`，只加载今日看板、今明日程、公告、活动规划文档和今日专注统计；进入日程时间轴或点击“加载完整待办 / 日程数据”时再读取完整数据
- 手机端主状态不再把完整专注记录和完整 AI 报告列表合并进普通每日看板 / 我的任务状态；今日专注统计改用 Room 聚合 Flow，AI 报告只在进入 `AI 报告` 页面时单独订阅
- 日历视图复用顶层事件日期索引，月视图、列表视图和可见全天日程不再各自重复按日期重建事件分组，降低日程数量变多后的重组开销
- 补充无 DDL 待办跨天单测，确保无 DDL 活动待办不是只在创建当天显示，而是每天都归入 `今日待办`
- 电脑端待办管理区文案改为“首屏优先加载每日看板、完整管理视图按需加载”，避免轻量数据模式下误导用户
- 版本号升级到 `1.9.13` / `versionCode 207`

## v1.9.12

- 修复 Android 今日看板小组件漏显示无 DDL 待办的问题：小组件的 board-range Room 查询现在会显式取出无 DDL 活动待办，和手机每日看板 / 桌面每日看板的“无 DDL 归入今日待办”语义一致
- 数据库版本升级到 `14`，新增 `todo_items` 看板待办、看板日程、活动提醒、分组 DDL、循环系列 DDL 复合索引，减少每日看板、小组件刷新、提醒扫描和分组/循环查询的大表扫描压力

- 规划台普通项目符号识别为无 DDL 待办时，预览候选会提示“普通项目符号已识别为无 DDL 待办”，降低把普通笔记误认为正式待办的理解成本
- 设置 -> AI 调用配置在编辑、启用/停用、排序、删除 AI 源后会尽量立即保存有效配置；如果仍有未保存或必填项缺失，会在页面内直接提示，降低填完 Base URL / API Key 后忘记保存的风险
- 设置 -> 电脑同步补充 5 分钟无授权连接自动关闭的说明，明确只打开网页但未输入正确密钥不算已连接
- 版本号升级到 `1.9.12` / `versionCode 206`

## v1.9.11

- 无 DDL 活动待办统一归入“今日待办”：手机每日看板、Android 小组件、电脑端每日看板和电脑端待办管理列表都会把它当作今天要处理的事项，不再把它塞到计划中 / 单独漂浮区
- 规划台本地解析器新增普通项目符号任务识别：`- 想办的事`、`* 整理材料`、`• 发消息` 会识别为无 DDL 待办，仍然支持写时间段时识别为日程
- 电脑同步开启后，如果 5 分钟内没有任何带正确访问密钥的电脑端 API 请求，会自动关闭电脑同步并停止前台服务，避免长期空跑
- 电脑同步连接地址按更可能可用的 Wi-Fi 地址优先排序，并在设置页说明 `10.*` / `172.*` 等地址通常来自 VPN、热点或虚拟网卡，不是 localhost
- 设置 -> AI 调用配置的 AI 源列表改成摘要卡 + 开关 + 更多菜单，避免 Base URL、模型、排序和删除操作挤在同一行
- 版本号升级到 `1.9.11` / `versionCode 205`
