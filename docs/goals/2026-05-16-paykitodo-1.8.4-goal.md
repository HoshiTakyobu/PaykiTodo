## Part 1：电脑端公告横幅长文本改为滚动

### 问题

1.8.3 的 `app.css:107-130` 公告横幅用 `text-overflow: ellipsis` 只截断不滚动。当用户写多条公告（用 ` · ` 串起）时，后面的公告永远看不到。

### 修复

在 `app.css` 中：

1. 横幅容器保留 `overflow: hidden` 和 `white-space: nowrap`，但去掉 `text-overflow: ellipsis`
2. `.announcements-banner span` 改为：
   ```css
   .announcements-banner span {
     min-width: 0;
     white-space: nowrap;
     animation: announcement-marquee 28s linear infinite;
     padding-left: 100%;
   }
   .announcements-banner:hover span { animation-play-state: paused; }
   @keyframes announcement-marquee {
     from { transform: translate3d(0, 0, 0); }
     to { transform: translate3d(-100%, 0, 0); }
   }
   ```
3. 文字短到不需要滚动时，避免空白滚动效果——在 `app.js:renderAnnouncements()` 中根据文本长度决定是否加 `marquee-needed` class，只有 class 存在时才应用 animation。判断阈值：文本字符长度 > 60 才滚动（粗略估计能装下中等屏幕一行的字数）

4. 鼠标悬停暂停滚动（`:hover` 上面已加），方便用户读完整条

### 验证

- 短公告（< 60 字）静止显示
- 长公告或多条公告横向滚动，鼠标悬停暂停

## Part 3：Widget 条目点击直达对应事项

### 问题

`TodoWidgetProvider.kt:60` 用了 `setPendingIntentTemplate(R.id.widget_list, openAppIntent)`，`TodoWidgetService.kt:60` 的 `setOnClickFillInIntent(R.id.widget_item_root, Intent())` 传的是空 Intent。结果点哪条 row 都只是开 App 主界面。

### 修复

1. `TodoWidgetProvider.updateWidget` 改用一个能接受 fill-in extras 的 PendingIntent：
   ```kotlin
   val templatePendingIntent = PendingIntent.getActivity(
       context,
       appWidgetId,
       Intent(context, MainActivity::class.java).apply {
           flags = Intent.FLAG_ACTIVITY_NEW_TASK or Intent.FLAG_ACTIVITY_CLEAR_TOP
           // 不设 data 和 extras，留给 fillInIntent 填
       },
       PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE or PendingIntent.FLAG_MUTABLE
   )
   views.setPendingIntentTemplate(R.id.widget_list, templatePendingIntent)
   ```
   注意：fillInIntent 模式下 PendingIntent 必须是 `FLAG_MUTABLE`（API 31+）。配合 `FLAG_UPDATE_CURRENT`。

2. `TodoWidgetService.getViewAt` 中根据 row 类型决定 fillInIntent 携带的 extra：
   ```kotlin
   val fillIn = Intent().apply {
       when (row.type) {
           WidgetRowType.TODO -> {
               putExtra(MainActivity.EXTRA_OPEN_TODO_ID, row.itemId)
           }
           WidgetRowType.EVENT -> {
               putExtra(MainActivity.EXTRA_OPEN_EVENT_ID, row.itemId)
           }
           WidgetRowType.ANNOUNCEMENT -> {
               putExtra(MainActivity.EXTRA_OPEN_PLANNING_NOTE_ID, row.sourceNoteId)
           }
           else -> {} // header / empty 不带 extra，回落到主界面
       }
   }
   setOnClickFillInIntent(R.id.widget_item_root, fillIn)
   ```

3. 给 `WidgetBoardRow` 加 `type: WidgetRowType` 和 `itemId: Long` / `sourceNoteId: Long` 字段。当前的 `WidgetBoardRow` 用 `stableId` 作为唯一标识，但负数表示 header / empty / announcement，混在一起。建议显式分开：
   ```kotlin
   enum class WidgetRowType { HEADER, EMPTY, TODO, EVENT, ANNOUNCEMENT }
   data class WidgetBoardRow(
       val stableId: Long,
       val type: WidgetRowType,
       val itemId: Long = 0,
       val sourceNoteId: Long = 0,
       // ... 现有字段
   )
   ```

4. `MainActivity` 接收 extra 后跳转：
   - `EXTRA_OPEN_TODO_ID`：跳到 ACTIVE section 并自动打开该 todo 的编辑器（如果该待办还存在）
   - `EXTRA_OPEN_EVENT_ID`：跳到 CALENDAR section 并自动打开该日程
   - `EXTRA_OPEN_PLANNING_NOTE_ID`：跳到 PLANNING section 并切换到对应文档

   现有 `DashboardLaunchRoute` 机制（`launchRoute?.settingsSectionKey`）可以扩展：

   ```kotlin
   data class DashboardLaunchRoute(
       val settingsSectionKey: String? = null,
       val targetTodoId: Long? = null,
       val targetEventId: Long? = null,
       val targetPlanningNoteId: Long? = null
   )
   ```

   `MainActivity.onNewIntent` 解析 extras 重新构造 launchRoute。

### 验证

- 点击桌面 widget 上某条 todo → 直接打开编辑器
- 点击某条日程 → 跳到日历该日程的预览
- 点击公告 → 跳到该公告所在规划文档
- 点击 header 或空状态 → 仅打开 App 主界面（落到默认看板）

## Part 6：检查启动动画 600ms 实际效果

### 背景

1.8.1 把 `delay(1600)` 改成 `delay(600)`。但 600ms 在低端机或冷启动时可能仍偏长，或者 600ms 后界面还没准备好导致白屏闪烁。

### 任务

1. 在 `DashboardScreen.kt:177` 附近确认：`launchVisible` 在 `delay(600)` 后才设为 false，但首屏的 LazyColumn 数据可能还没从 ViewModel 流回来
2. 改为更智能的：
   ```kotlin
   LaunchedEffect(uiState.todos.isNotEmpty() || uiState.dataReady) {
       if (uiState.dataReady) {
           launchVisible = false
       }
   }
   LaunchedEffect(Unit) {
       delay(800)  // 兜底：最多 800ms 后无论如何都隐藏
       launchVisible = false
   }
   ```
   `TodoUiState` 增加 `dataReady: Boolean` 字段，在 ViewModel 第一次 emit 真实数据时设为 true

3. 这样冷启动数据加载快时 < 600ms 就能消失，慢时也最多 800ms

如果发现现在的 `delay(600)` 体感正常，本项跳过。
