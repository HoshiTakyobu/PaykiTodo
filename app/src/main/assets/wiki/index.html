PaykiTodo 使用说明

待办 强提醒 日历 倒数日 批量导入 模板与备份

这份说明的目标不是告诉你“软件大概能做什么”，而是告诉你“某个按钮点进去到底怎么用、会发生什么、遇到什么边界情况是正常的”。如果你对软件中的某个操作有疑惑，应该能在左侧目录里找到对应入口。

推荐阅读顺序：第一次上手 → 首页与待办列表 → 全屏提醒界面 → 日历界面总览 → 批量导入语法 → 提醒权限、诊断与提示音。

待办系统

适合做需要强提醒、需要全屏弹出、需要完成/取消/延后的事情。

日历系统

适合课程表、值班表、全天日程、跨天安排和时间轴规划。

诊断与备份

提醒链路测试、崩溃日志、导出 JSON、自动备份都在软件内。

当前版本最重要的交互原则

  • 活动态只有三种：已错过、今日待办、计划中。其中 已错过 仍然是活动态，不算历史态。
  • 真正进入历史记录的只有两种状态：已完成 和 已取消。
  • 无 DDL 的待办可以存在，会统一显示在今日待办中；它不会启用提醒，也不会启用循环。
  • 倒数日不是独立事项，而是待办 / 日程上的显示开关：待办按 DDL 计算，日程按开始时间计算。
  • 循环任务/循环日程在编辑、取消、删除时，常常会先要求你选择影响范围。
  • 提醒到点后的主目标是让你明确操作，不再保留“忽略一下先关掉”的入口。

第一次上手

如果你刚装好软件，建议先按下面的顺序做一轮完整配置。这样后续测试提醒时不会被系统权限或后台限制误伤。

  1. 进入 设置 → 提醒权限，把通知、精确闹钟、全屏提醒、免打扰穿透、忽略电池优化、辅助功能提醒都检查一遍。
  2. 进入 设置 → 提醒链路诊断，先跑一次短延迟测试，确认通知、前台服务、全屏页、锁屏显示链路至少能通。
  3. 进入 设置 → 提示音，可在 内置提醒音 和 系统通知提示音 之间切换。
  4. 回到首页，先建一条有 DDL、带提醒的普通待办，再建一条无 DDL 的随手任务，对比两种行为差异。
  5. 打开日历，先手动创建一个普通日程，再试一次批量导入，熟悉两种录入方式。
  6. 最后到 设置 → 数据与备份 里选一个备份目录，避免后续卸载应用导致本地数据直接丢失。
PaykiTodo 目前仍然是本地单机优先。卸载应用会清掉应用私有目录里的数据库、设置和语录缓存。要保数据，必须主动备份或导出。

输入框旁边的问号是什么

提醒时间、批量导入、自定义延后等输入框旁边会出现一个圆圈里的问号。点一下会弹出当前输入框的语法说明和示例，不需要你记住所有规则。

写法含义

5,15提前 5 分钟和 15 分钟分别提醒。编辑待办/日程时可多选。

抽屉菜单与导航

点左上角三道杠会打开抽屉菜单。抽屉不是全屏覆盖，而是一个左侧面板，主要用于切换模块。分组筛选和分组管理已经收口到待办页顶部，不再占用抽屉空间。

顶部区域

  • 最上方显示软件图标和 PaykiTodo 名字。
  • 底部有显示模式切换按钮。点击后会弹出 浅色、深色、跟随系统 三种模式。
  • 如果当前是 跟随系统，图标会根据系统当前明暗显示为太阳或月亮。

菜单结构

入口作用
待办进入待办页。页面顶部有横向分组 chip，可多选交集筛选、创建、重命名、改色或删除分组。
日历进入时间轴/周视图/月视图/列表视图所在的日历模块。
历史记录查看已完成和已取消的任务。
设置目录式设置页。大部分调试与权限项都在这里。
关于版本号、内部版本、包名、项目信息和说明入口。

分组筛选是怎么生效的

  • 进入 待办 后，顶部的分组 chip 会显示 全部、各个分组和 新建。
  • 点击某个分组后，待办页仍然保持 已错过 / 今日待办 / 计划中 这三个板块。
  • 变化的是内容过滤：一个分组时只显示该分组内的任务；多个分组同时选中时，只显示同时属于所有选中分组的任务。
  • 再次点击已选分组会取消该分组；点击 全部 会清空所有筛选；长按分组 chip 可编辑名称、颜色或删除自定义分组。

首页与待办列表

首页是 PaykiTodo 的待办主战场。它保留语录、分区、预览卡片和直接勾选完成的能力。

语录卡片

顶部显示一句本地缓存语录。右侧刷新按钮可以切到下一条，退出后会记住你停留的位置。

已错过

显示第一天及以前已过 DDL 但未完成、未取消的任务。默认展开；如果没有错过任务则自动隐藏。

今日待办

显示今天应处理的任务。默认展开。

计划中

显示未来有 DDL 的任务。无 DDL 任务会统一放入今日待办，避免想办的事被藏到未来列表。

任务排序规则

  • 已错过：按时间从早到晚排。
  • 今日待办：按今天内的时间从近到远排；无 DDL 任务也会放在这里，通常排在有具体时间的任务之后。
  • 计划中：只显示未来有 DDL 的任务。

任务卡片怎么看

  • 左侧竖条颜色来自任务的主分组，也就是编辑器里第一个选中的分组。
  • 标题前的小标签会显示这个任务所属的分组；一个任务可以同时属于多个分组。
  • 有备注时，卡片会直接展示备注摘要。
  • 有 DDL 时，会显示 ⏰ DDL。
  • 循环任务会额外显示 循环任务 字样。
  • 错过任务的标题、备注、DDL 会偏红，便于快速识别。

快速完成

点卡片最左侧复选框即可完成任务。完成时会有划线和烟花过渡动画，然后任务会移动到历史记录。

倒数日

  • 在待办或日程编辑器里打开 倒数日 后，它会出现在每日看板的倒数日卡片里。
  • 待办倒数日按 DDL 时间计算；日程倒数日按开始时间计算。
  • 倒数日行按真实剩余时长显示：1 天以上显示 9d + 20h 36m，不足 1 天显示 23h + 15m，不足 1 小时直接显示 15m；不再显示秒，目标过点后会自动隐藏。
  • 无 DDL 待办不能作为倒数日目标。它可以留在今日待办，但没有可计算的截止日期。

看板公告和桌面小组件

  • 公告现在直接写在规划台里，不再放在设置页。任意未归档规划文档中写 #公告 5.16-7.1 公告内容、- [ ] #公告 5.16 公告内容 或 > [!公告] 公告内容 即可。
  • 公告可以多条同时生效；今天在日期范围内时，每日看板顶部会显示橙色公告横幅，过期、未到期、归档或删除的公告会自动隐藏。
  • 规划台预览模式下，公告行会显示橙色“全局公告”提示；电脑端网页控制台也会在顶部显示当前生效公告，并跟随系统深色模式。
  • 公告文字较长时会在横幅里横向滚动，适合放“这段时间禁止某项娱乐”这类自我约束提醒；电脑端横幅在 60 字以内保持静止，长文本滚动时可悬停暂停。
  • Android 桌面长按空白处添加小组件时，可以选择 PaykiTodo 的 今日看板 小组件。它会展示公告、今日待办、今日日程和明日日程摘要，也不再把倒数日嵌进今日看板小组件。
  • 小组件采用接近每日看板的卡片式视觉：外层叠加每日看板背景图和遮罩，列表直接从橙色公告横幅、问候卡、分组色条待办卡，以及合并今天 / 明天内容的一张日程看板卡开始，不再显示三横线按钮、“每日看板”标题或日期头部。当前默认建议尺寸更偏 4x5 竖向看板，添加页预览也使用静态看板布局；待办行有分组色条、复选框视觉、分组标签、备注摘要和 ⏰ DDL HH:mm 文案，日程行尽量保持 App 内看板的透明行 + 竖色条节奏，进行中的日程会显示金色边框和淡金色底，弱化 RemoteViews 系统列表感。
  • 无 DDL 的活动待办也会进入小组件的“今日待办”。它们不会启用提醒或循环，但不会因为没有 DDL 而被桌面小组件漏掉。
  • 桌面还可以单独添加 PaykiTodo 的 倒数日 小组件。它不显示额外标题 / 日期 / 项数头部，只用可滚动列表展示倒数目标；待办目标保留完成圆圈，日程目标不显示圆圈，点击目标行会直接进入对应待办或日程详情。
  • 小组件支持横向和纵向拖动调整尺寸；尺寸越大，列表能露出越多看板内容。点击“今日待办”区块标题会进入应用内 待办，点击具体待办行会打开该待办详情预览；点击“今日日程”区块标题或日程聚合卡会进入 日历，点击具体日程行会打开该日程详情 / 日历预览；点击公告行会进入来源规划文档，空状态行会回到默认每日看板。小组件颜色会适配系统深色模式，浅色模式下卡片和文字对比度更高；日期变化、手动改时间、时区变化和应用替换后会主动刷新列表，避免跨日后列表内容停留在旧日期。小组件选择器里会显示 PaykiTodo 每日看板、倒数日的名称、说明、建议尺寸和静态预览。

创建与编辑待办

首页右下角的加号默认用于新增待办。编辑时则从任务卡片预览窗里的 修改 按钮进入。

备注可为空。首页卡片最多展示摘要，完整内容在预览和提醒页查看。
所属分组可多选。第一个选中的分组决定主颜色，全部选中分组都会作为标签和筛选归属保存。

无 DDL 任务

  • 把 设置 DDL 关掉即可创建无截止日期任务。
  • 无 DDL 任务会统一排在 今日待办 中，适合先记录“今天想办但还没定时间”的事项。
  • 无 DDL 任务不会启用提醒，也不会启用循环，也不能作为倒数日；如果需要强提醒或倒数显示，请补上 DDL。

循环任务开关

任务层面不额外区分“任务类型”，而是通过 循环任务 开关决定当前待办是否按循环方式生成多个实例。

目前支持的循环规则

  • 每天
  • 每周的周几，支持多选
  • 每月的第几个星期几
  • 每月的 D 日
  • 每年的 M 月 D 日

循环截止日期

  • 循环任务必须设置截止日期。
  • 默认值是首个任务日期后的 90 天。
  • 可以手动修改。

每月 D 日的特殊规则

  • 如果你选的是 29、30、31 这类日期，而某个月没有这一天，则自动落到该月最后一天。
  • 例如选择每月 31 日，到了 4 月会生成在 4 月 30 日。
  • 编辑器下方会显示一行红色小字提示这个规则。

预览循环生成

点 预览循环生成 可以先看这个循环配置大概会生成多少条实例，以及前几条的实际落点。建议你在课表、吃药、值班这种重要循环项上先预览再保存。

编辑循环任务时的范围选择

如果你修改的是仍处于活动态的循环任务，系统会先弹出“选择修改范围”。

范围含义
当前事件只改当前这一条实例。
当前事件和此后所有事件从当前实例开始，当前和未来一起改。
所有事件整个系列全部改。

待办预览、完成、取消、恢复

点击任务卡片主体会打开预览窗；左侧圆圈只负责完成待办，避免误触。很多不适合直接暴露在列表里的按钮，现在都放在预览窗里了。

活动任务预览窗

  • 显示分组、标题、备注、创建时间、DDL。
  • 底部按钮一般是 取消待办（归档）、删除、修改。
  • 取消待办（归档） 会停止提醒并进入历史记录；这不是删除。
  • 删除 会直接移除待办，不进入历史记录，也无法恢复。
  • 如果是循环任务，点 取消、删除 或 修改 前通常会先要求选择范围。

取消任务

  • 取消后任务会停止提醒，并进入历史记录。
  • 一次性任务通常直接取消当前这条。
  • 循环任务会先要求选择作用范围。

删除任务

  • 删除是硬删除，不进入历史记录。
  • 循环待办选择“仅当前”删除时，系统会记录这一次已跳过；它不会出现在历史记录里，也不会被循环模板重新生成。
  • 循环待办选择“当前及之后”或“全部”删除时，会直接删除对应范围内的实例和循环模板范围。

历史任务预览窗

  • 历史列表中的卡片只显示分组、标题、备注，保持界面简洁。
  • 更多细节例如创建时间、DDL、完成/取消时间，放在预览窗里查看。
  • 历史预览窗里可以看到 恢复 和 修改。

恢复任务

历史记录中的任务可以通过 恢复 回到活动态。恢复后是否继续参与提醒，要看当前任务本身的时间条件和状态，而不是无条件重新响铃。

错过任务是怎么产生的

  • 任务过了 DDL 后，如果在短缓冲窗口内仍未完成、未取消、未改期，它会进入 已错过。
  • 进入 已错过 后仍然是活动态，不会直接丢去历史。
  • 你仍然可以点开它进行修改、完成或取消。

规划台

规划台是 PaykiTodo 的“前置思考区”。它不是替代待办和日历，而是让你先像写备忘录或 Obsidian 文档一样，把接下来要做的事、粗略时间、DDL 和拆分任务写下来。现在的大纲输入会先形成草稿节点，只有你显式发布后，才会进入真正会提醒你的待办或日程。

规划台当前不做 AI 自动规划、拖拽排期、甘特图或复杂项目树。它先解决最实际的问题：自由写计划 → 草稿沉淀 → 人工检查 → 发布到待办 / 日程。Markdown 兼容区仍保留“识别 → 预览 → 导入”的旧批量流程。

如何进入

  1. 打开左侧抽屉菜单。
  2. 点击 规划台。
  3. 如果还没有文档，系统会自动创建 我的规划。
  4. 再次进入规划台时，会默认打开你最后一次使用的文档。

规划文档怎么管理

  • 文档：打开已有规划文档列表，点击即可切换。
  • 新建：创建一篇新的规划文档。
  • 自动保存：手机端正文会自动保存；电脑端也可用 Ctrl+S 手动保存。
  • 重命名：修改当前文档名称。
  • 归档：从当前文档列表中隐藏这篇文档。
  • 删除：删除前会询问确认，删除后无法恢复，建议谨慎使用。

手机端快捷栏怎么用

手机端规划台默认是大纲式连续输入：底部空行写完按回车，会生成一条草稿节点。草稿只留在规划台里，不会进入待办、日历、每日看板或小组件；确认要执行时，点节点右侧纸飞机发布，或点顶部 发布N条 批量发布。

手机中文输入时手打缩进和公告标记比较麻烦，所以规划台只保留少量高频快捷操作。为了最大化书写区域，快捷操作默认收起；普通待办、DDL、提醒、分组和日程建议直接自然书写，发布时再按规则解析。

按钮作用
子任务在当前行下面新起一行，插入缩进后的   - [ ] 。
公告在新行插入 #公告 ，继续补日期范围和公告正文后，会显示到每日看板、电脑端和桌面小组件。

- [ ] 整理保研材料
- 想办但还没定时间的事
* 再整理一下资料
• 给导师发消息
- [ ] 写课程论文 #ddl 5.28
- [x] 已经完成的事情
  - [ ] 整理获奖证明 #ddl 5.20 #group 保研
  • - [ ] 会识别为待办候选。
  • 普通项目符号 - / * / • 也会识别为无 DDL 待办，并统一显示在今日待办中。
  • - [x] / - [X] 默认跳过，不会重复导入。
  • 在预览状态点击 checkbox，会把原始 Markdown 行在 - [ ] 和 - [x] 之间切换；这只修改规划文档，不会直接完成已经导入的正式待办。切回编辑时会尽量定位到刚刚操作的原文行。
  • 缩进子任务第一阶段会作为独立待办导入，并在备注里记录所属大任务。

怎么写自然日程

10:00-12:30 作业1
10:00~12:30 作业1
10:00 至 12:30 作业1
明天 19:30-21:00 整理保研材料
5.28 14:30-16:00 写论文
复习 14:00-16:00
5/28 下午 2:30～下午 4:00 小组讨论
5月28日 11:55pm-次日00:30 整理材料
10:00-12:00, 【课程】习思想，"@主楼B1-412"
  • 行里有明确时间段时，默认识别为日程，不要求时间段必须在行首。
  • 如果同一行显式写了 #ddl，会优先按待办处理；例如 会议 9:00-10:00 讨论 #ddl 5.28 是一个带 DDL 的待办，不是日程。
  • 支持 -、~、～、至、到 等常见时间段分隔符。
  • 逗号分隔的轻量日程可按“时间段，标题，地点”理解；"@主楼B1-412" 中的引号只用于保护地点文本，导入时不会保存外层引号。
  • 没有写日期且没有日期标题上下文时，默认今天，预览里会标出。
  • 导入日程时，默认只创建日程；如果你确实想把日程结束时间也作为一个 DDL，可以在预览里手动勾选“同步创建以日程结束时间为 DDL 的待办任务”。

日期上下文标题

# 明天

- [ ] 09:00-10:30 写论文
- [ ] 14:00-16:00 复习

标题可以写 # 今日计划、# 今天、# 明天、# 后天、# 周五、# 周五计划、# 5.28、# 5/28 周末计划、# 5/28周末计划、# 5月28日、# 2026-05-28。下面没有写日期的日程会继承最近的日期标题。# 收集箱 只是普通分区，会清空之前的日期上下文。

DDL 支持哪些格式

#ddl 后面可以写这些常见格式：

#ddl 5.28
#ddl 05.28
#ddl 5/28
#ddl 5-28
#ddl 5月28日
#ddl 2026-05-28
#ddl 2026/05/28
#ddl 2026.05.28
#ddl 2026年5月28日
#ddl 明天
#ddl 今天 16:30
#ddl 下午 2:30
#ddl 5.28 23:59
#ddl 5.28,23:59
#ddl 5月28日 11:55pm
  • 只写日期、不写时间，默认当天 23:59。
  • 只写时间，例如 #ddl 16:30，默认今天。也可以快速写 任务M ddl 15:00，系统会按今天 15:00 的待办处理。
  • 日期和时间之间可以用空格，也可以用中文逗号，例如 #ddl 5.28，23:59。
  • 斜杠日期和常见全角分隔符也支持，例如 #ddl 5/28、#ddl 5．28。
  • 如果解析出的 DDL 已经过期，预览会提示风险，不会偷偷改到明年。

提醒和分组

- [ ] 交材料 #ddl 5.28 #remind 5,15 #group 保研
- [ ] 下午找老师 #ddl 5月28日 18:00 #remind 2:30 pm
- [ ] 准备材料 #ddl 2026-05-30 18:00 #remind 5/30 下午 2:30
  • #group 保研 会尝试映射到现有分组。
  • #remind 5,15 表示提前 5 分钟和 15 分钟提醒。
  • #remind 16:30 表示 DDL / 日程开始当天 16:30 提醒；也支持 2:30 pm、下午 2:30、5.30 17:30、5/30 下午 2:30、5月30日 17:30。
  • #remind 5.30,17:30 或 #remind 5月30日，17:30 会按一条具体提醒时间识别。
  • 没写 #remind 时，规划台导入默认提前 5 分钟、全屏提醒、响铃 + 震动。

哪些标签暂时不当语法

#today、#tomorrow、#important、#project 目前不会写入正式待办属性，也不会自动安排日期。它们会保留在标题里，避免用户误以为已经生效。当前真正会被解析的主要是 #ddl、#remind、#group、#schedule 和 #task。

识别、预览和导入

  1. 日常建议直接在大纲里一行一行写，先得到草稿节点，再发布。
  2. 如果要粘贴旧 Markdown 或大段文本，再使用 Markdown 兼容区。
  3. 点击 识别。
  4. 在预览中检查每条内容的类型、标题、DDL 或起止时间、提醒、分组、地点和是否同步创建待办。日程默认只创建日程，只有手动勾选“同步创建以日程结束时间为 DDL 的待办任务”才会额外生成待办。
  5. 如果识别结果不符合预期，可以在预览卡片里直接修改标题、分组、备注、DDL、日程起止时间和提醒输入；DDL 和起止时间也支持 5.28 23:59、明天 16:30、下午 2:30 这类自然写法，非法提醒会标红并阻止导入。
  6. 可用 全选可导入项 / 全不选 快速控制候选；取消不想导入的条目。
  7. 没有选中任何有效候选时，导入按钮不可用。
  8. 点击 导入选中。这是 Markdown 兼容区的正式导入流程；大纲草稿则通过纸飞机或 发布N条 发布。

#imported 是什么

如果一行已经带有 #imported，规划台会默认跳过它，避免重复导入。导入成功后，规划台会自动给被导入的原文行追加 #imported，并立即保存当前规划文档。

规划台公告

公告是每日看板和桌面小组件上方反复出现的自我提醒。它不再从设置页维护，而是直接写进规划台文档，便于和阶段计划放在一起。

  • #公告 5.16-7.1 期间禁止游玩舞萌DX游戏
  • - [ ] #公告 5.16 今天必须看见的提醒
  • > #公告 5.16 引用样式提醒
  • #公告 2026-05-16 2026-05-20 本周推进保研材料
  • > [!公告] 长期提醒：先完成今天三件大事

不写日期时视为长期公告；写日期范围时按今天是否落在闭区间内决定显示。归档、删除文档或删除公告行后，公告会从看板、电脑端网页横幅和小组件里消失。尾部 #imported 或 #group 这类标签不会显示到公告正文里。

电脑端规划台

开启电脑同步后，电脑网页控制台左侧有 规划台 Tab。电脑端大纲事项支持文档选择、新建、删除、自动保存、草稿显示、单条发布和批量发布；新增大纲行默认是草稿，不会立即进入待办或日历。Markdown 兼容编辑区仍支持识别、预览内编辑和导入选中；预览内可以编辑标题、分组、备注、DDL、日程开始 / 结束时间、地点、全天、倒数日、重复规则和提醒时间。日程默认只创建日程，只有你手动勾选同步待办时，才会额外创建一个 DDL 等于日程结束时间的待办。电脑端日程新增窗口也支持 每周多时间段：开启后为同一标题填写多个周几和起止时间，确认时批量创建同名每周循环日程。新建规划文档、刷新已导入项、顺延规划事项和新增子项都使用 PaykiTodo 自己的应用内弹窗；顺延时可以直接下拉选择具体事项和影响范围，不需要记序号或输入 1/2/3。电脑端支持 Ctrl+S 保存、Ctrl+Enter 识别；电脑端和手机端操作的是同一份本地数据库。生效的规划台公告会显示在电脑端网页顶部横幅，长公告会滚动且可悬停暂停，网页整体会跟随系统深色模式。

如果使用 AI 识别，AI 返回的候选也会先进入预览再导入。明确的 Markdown / DDL / 时间段语法会先由本地规则处理；当本地规则已经覆盖全部有效行时，不会额外调用 AI，避免清晰写法被模型改错。电脑端现在可以直接导入 AI 候选，不再因为 AI 候选 ID 和本地规则 ID 不一致而出现“识别到 1 条，导入 0 条”。分组采用保守规则：只有原文显式写了 #group、分组：、项目： 或 课程： 时才会保留分组；普通标题不会被 AI 擅自拆出分组。

AI 识别配置

进入 设置 → AI 调用配置 后，可以填写规划台 AI 识别要用的服务名称、Base URL、API Key 和模型名。Base URL 可以填服务根地址、到 /v1、完整 /v1/chat/completions，或完整 /v1/models。编辑单个 AI 源时，可以先只填 Base URL 和 API Key，点 获取模型 从 OpenAI-compatible /models 接口拉取可用模型并下拉选择；如果服务不支持模型列表，仍可手动填写模型名。点 测试连接 会用当前模型测活，成功会显示响应字数，失败会显示 HTTP、超时或非 JSON endpoint 错误。启用后，手机端和电脑端规划台点 识别 会先运行本地规则；明确 Markdown、DDL 或时间段语法已经覆盖全部有效行时不会调用 AI，存在未覆盖的自然语言时才按列表顺序调用可用 AI 源。AI 失败、超时或配置不完整时会自动回到本地规则。AI 结果也必须先进入预览确认，不会直接写入正式待办或日程；编辑、换行和自动保存不会自动调用 AI。AI 不会从普通标题里自动猜分组，例如 16:05-18:00 入党表格填写 会保持标题为“入党表格填写”、分组为空，除非你明确写出分组标记；@图书馆3楼、"@主楼B1-412" 会进入地点字段，不会留在标题里。

AI 源列表使用摘要卡展示。完整配置的 AI 源在添加、编辑、启用/停用、排序、删除后会尽量立即保存；如果启用了不完整的源，页面会直接提示缺少 Base URL、API Key 或模型名，避免填完后忘记保存。

AI 日报 / 周报

设置 → AI 调用配置 里还有 AI 日报 / 周报 区块。你可以启用每天定时日报、每周日周报，使用 HH:mm 格式设置生成时间，并通过 报告保留时长 选择 30 天、90 天、365 天或永久保存；也可以点击 了解 AI 日报 查看分卡片使用指引，或点击 立即生成一次日报 进行调试。准点生成依赖系统的「精确闹钟」权限；如果没有开启，PaykiTodo 会降级使用系统允许的后台调度，但可能被系统延后。

  • 日报会汇总今天完成待办、错过待办、今日日程、明日日程和明天 DDL。
  • 周报会汇总本周完成 / 错过待办、本周日程和下周 DDL。
  • 启用 AI 源时，报告会优先调用规划台 AI 多源轮询；AI 不可用、配置失败或网络失败时，会用本地模板生成简短报告。
  • 日报和周报都会写入独立的 AI 报告 归档入口（抽屉 → AI 报告），不再混入规划台文档列表。
  • 生成新报告后，会按你设置的保留时长清理旧报告；选择永久时不会自动删除归档。
  • 生成后会发送低优先级通知；点击通知会直接打开对应报告详情。Android 13+ 如果没有通知权限，则只写入归档、不弹通知。
  • AI 报告 支持关键词搜索、全部 / 日报 / 周报类型筛选、全部时间 / 近 7 天 / 近 30 天 / 近 90 天范围筛选；点击卡片看完整正文，长按卡片或详情内删除按钮会先弹出确认。
真实 API Key 只保存在本机设置中。备份 JSON 会保留服务名称、Base URL 和模型名，但不会导出 API Key；导入备份也不会把本机已有 Key 清空。电脑同步开关状态和访问密钥同样不写入备份，恢复备份后电脑同步默认关闭，需要手动开启并以设置页显示的新密钥为准。

电脑同步

从 1.6.0 开始，PaykiTodo 内置了一个局域网网页控制台。手机和电脑在同一局域网时，电脑浏览器可以直接连接手机，查看并修改手机里正在使用的同一份待办和日程数据。

这不是单独的 Windows 客户端，而是手机端开出一个网页控制台。优点是不用再装一个桌面程序，电脑打开浏览器就能用。

电脑端现在能看到什么

  • 第一个入口是 每日看板，不是单纯的待办管理后台。
  • 看板顶部会显示今天日期和当前正在进行 / 下一项。
  • 下方会显示倒数日、今日待办、今日日程、明日日程；已经结束的今日日程不会再计入“今日日程”数量，正在进行的日程会有金色高亮。
  • 待办管理列表仍保留在每日看板下方，用于管理已错过、今日、计划中和历史记录；无 DDL 待办统一归入今日待办。
  • 待办管理列表按页读取，支持搜索标题、备注和地点；点击 加载更多 会追加下一页，避免大量历史待办一次性拖慢电脑端。

入口在哪里

  1. 进入 设置 → 电脑同步。
  2. 打开 启用电脑同步 开关。
  3. 记下页面里显示的 访问密钥。
  4. 点 显示连接地址 后，优先使用页面推荐的 Wi‑Fi 地址，例如 http://192.168.1.23:18765；如果刚打开开关后短暂显示“正在启动”，等几秒再看地址，仍无地址时关闭再重新开启电脑同步。
  5. 在电脑浏览器里打开这个地址。
  6. 在网页左侧输入访问密钥，点 连接手机。

如果开启电脑同步后 5 分钟内没有任何电脑端用正确访问密钥完成连接，手机端会自动关闭电脑同步并停止前台服务，避免长期无用空跑。只打开网页但没有输入正确密钥，不算已连接。

页面可能同时显示 192.168.*、10.*、172.* 等多个地址，因为 Android 会枚举所有非回环 IPv4 网卡；这些地址可能来自 Wi‑Fi、VPN、热点、虚拟网卡或运营商/校园网适配器。localhost 实际是 127.0.0.1，不是这些地址。一般优先试 192.168.*，连不上再试其它地址。

当前已经支持什么

  • 查看手机中的每日看板、分页待办管理列表和日程时间轴。当前版本的电脑同步服务会限制并发连接数量，长规划文本会按 UTF-8 字节完整读取，减少电脑端保存中文长文档时的截断风险。
  • 点击待办卡片会先打开详情预览；预览里可以完成、取消待办（归档）、删除或进入编辑。点击日程卡片会直接进入编辑器，删除仍需要确认。
  • 新增 / 编辑待办，支持 DDL、提醒时间、提醒方式、多分组、循环、倒数日、铃声和震动；电脑端会保留手机端已有的全部分组标签。
  • 新增 / 编辑日程，支持起止时间、全天、地点、备注、颜色、倒数日、打卡追踪、提醒方式、循环和每年同农历月日。
  • 在日程时间轴顶部使用 周历 Excel 导入：手动选择电脑本地的 review.xls / .xlsx 后，网页会只读解析表头日期、左侧节次时间和单元格内容，生成候选日程预览。单元格内写了显式时间段时优先使用单元格时间，合并单元格会按合并区域末行推断结束时间。
  • 周历导入不会修改 Excel、不会调用 存个档.bat、不会归档历史表格、不会设置桌面壁纸；只有你在预览中勾选并确认导入后，才会向手机端新增日程。已经存在或 Excel 内重复的候选会自动标记为重复并跳过。
  • 每日看板里的日程会打开详情预览；如果该日程开启了打卡追踪，电脑端可以查看打卡记录、总投入时间，并执行 签到 / 签退。日程时间轴中的日程仍直接进入编辑器，便于快速改时间。
  • 所有改动直接写入手机本地数据库，并复用手机端现有的提醒调度逻辑。

当前限制

  • 第一版优先保证同局域网直连，不是公网云同步。
  • 电脑端仍是浏览器控制台，视觉和交互会继续向手机端靠拢，但还不是完整独立桌面客户端。
  • 浏览器安全限制下，网页不能绕过文件选择框自动读取 G:\PlanGit\review.xls。周历导入当前需要你手动选择 Excel 文件。
  • 模板操作和部分更细的日历交互仍建议在手机端做。
  • 如果手机没有连 Wi‑Fi，设置页里可能看不到可访问的 IPv4 地址。

它是怎么同步到手机的

网页控制台操作的不是一份“另外的副本”，而是手机应用当前正在使用的 Room 数据库。也就是说，电脑上新增的待办和日程，会直接出现在手机端；手机端完成或删除的内容，下次网页刷新时也会同步看到。

安全性说明

  • 网页接口要求你输入 访问密钥 才能操作。
  • 电脑同步开关状态和访问密钥只保存在本机设置中，不会写入备份 JSON；恢复备份后电脑同步默认关闭，需要手动重新开启，并以手机设置页显示的新密钥为准。
  • 如果怀疑密钥泄露，可以在设置页里点 重新生成访问密钥。
  • 建议只在你信任的局域网里开启这项功能。

分组筛选与管理

分组既是视觉颜色来源，也是筛选维度。现在分组不再作为抽屉里的独立页面，而是放在 待办 页顶部的横向 chip bar 中。同一个待办可以同时属于多个分组。

你能做什么

  • 点击 全部 查看所有活动待办。
  • 点击某个分组 chip，只看该分组下的活动待办；点击多个分组时按交集筛选，只显示同时属于所有选中分组的待办。
  • 再次点击已选分组会取消它；点击 全部 会清空所有分组选中状态。
  • 点击 新建 创建自定义分组。
  • 长按某个分组 chip，修改分组名称或颜色。
  • 长按自定义分组 chip 后可以删除该分组。

  • 默认分组不能直接删。
  • 仍被活动任务使用的分组不能直接删，需要先迁移或清空相关任务的多分组标签。

全屏提醒界面

当待办或日程到点后，如果触发的是全屏提醒模式，你会看到一个独立的全屏界面，而不是仅靠通知栏轻飘飘地提醒一下。

这个界面的目标

  • 尽量在锁屏、后台、解锁后都把界面拉到前台。
  • 不允许简单按返回键就关掉提醒页。
  • 要求你明确作出一个动作，而不是继续拖延。

待办提醒会出现哪些按钮

  • 我已完成
  • 延后 5 分钟
  • 延后 10 分钟
  • 取消任务
  • 自定义延后提醒，可输入分钟数，也可输入未来具体时刻，例如 16:30；目标时间不再限制 180 分钟内，并且只改下一次提醒，不改 DDL。
  • DDL 推迟，可输入 30分钟、往后推45分钟、16:30 或 2026-05-22 16:30。分钟数必须大于 0，具体时刻必须晚于原 DDL。

日程提醒和待办提醒的区别

  • 日程提醒的主按钮是 我知道了。
  • 日程提醒不需要“继续延后再等你做”的那套逻辑，所以不会和待办完全一样。
  • 日程会尽量把 日期 和 时间段 直观显示出来。

延后时间的对齐规则

延后不是简单的“当前时间 + N 分钟 + 保留秒数”，而是按分钟对齐。也就是你在任意秒数点下延后后，新的提醒时刻会把秒和毫秒清零。

例如现在是 11:05:38，你点 延后 1 分钟，预期时刻应该是 11:06:00，而不是 11:06:38。

循环任务在提醒页取消时的范围选择

如果当前提醒对应的是循环任务，按 取消任务 后会先弹出范围选择：

  • 当前事件
  • 当前事件和此后所有事件
  • 所有事件

AI 调用配置

这里用于填写规划台 AI 识别要用的服务名称、Base URL、API Key 和模型名。Base URL 可以填服务根地址、到 /v1、完整 /v1/chat/completions，或完整 /v1/models。单个 AI 源编辑弹窗里有 获取模型 和 测试连接：获取模型会用 Base URL / API Key 调用 /models 并显示模型下拉，失败时仍可手动填写模型；测试连接用于保存前检查当前 Base URL、API Key 和模型是否能正常返回。如果服务返回 HTML 或网页内容，会提示检查 Base URL。启用后，手机端和电脑端规划台“识别”都会先用本地规则处理明确语法；只有仍有未覆盖自然语言时，才按列表顺序调用可用 AI 源。如果 AI 不可用，会回到本地规则识别。AI 日报 / 周报也复用这里的 AI 源，失败时会用本地模板兜底；设置卡片里的 了解 AI 日报 会解释启用步骤、独立 AI 报告 归档入口、通知跳转和常见问题。真实 API Key 只保存在本机设置中，不会写入备份 JSON，导入备份也不会清空本机已有 Key。

提醒声音策略与提示音

  • PaykiTodo 现在把“音色来源”“播放通道”和“App 内部音量”分开配置。
  • 播放通道可以选 闹钟通道、无障碍辅助通道、通知通道 或 媒体通道。
  • PaykiTodo 内部音量 只控制软件自己播放提醒音的音量，不会新增系统级 PaykiTodo 独立音量滑块。
  • 工作模式 开启后默认不外放提醒声音，并强制使用更强震动，即使单条事项原本关闭了震动，也会避免静音无感；日程提醒也会走全屏 / 无障碍兜底链路。
  • 提醒时临时提升系统通道音量 会临时修改所选系统通道的全局音量，播放结束或提醒结束后尽量恢复原值。
  • 播放通道、周起始日和日历默认提醒方式采用下拉选择；音量百分比采用滑条与数值输入，避免按钮堆叠占用空间。
  • 设置 → 提示音 → 内置提醒音：恢复软件自带音效。
  • 设置 → 提示音 → 系统通知提示音：打开系统通知提示音列表，把它作为提醒音色来源。
临时提升系统通道音量不是 PaykiTodo 私有音量。它会短时间改变系统对应音量通道，因此默认关闭，建议只在明确需要时开启。

  • 新增日程时可以打开 每周多时间段，适合课程、实验、固定值班等同一事项每周有多个时间段的情况。

  3. 确认无误后再点 一键导入。

备份与恢复

PaykiTodo 当前不是云同步产品，所以备份是你避免数据丢失的第一道保险。

入口

进入 设置 → 数据与备份。

你会看到哪些按钮

按钮作用
选择备份目录选择自动备份写入的位置。

自动备份 开关开启后，任务、模板或分组发生变动时会尝试写入备份目录。

备份里包含什么

  • 任务实例
  • 循环模板
  • 分组
  • 提醒链路诊断日志
  • 周模板 / 学期模板

导入会怎样

导入目前更偏“整库恢复”思路，而不是复杂合并。也就是说，如果你准备从一个旧 JSON 恢复，请先确认现有数据是否已经另外导出保存。

常见问题

1. 为什么我创建了无 DDL 任务，却没有提醒？

这是预期行为。无 DDL 任务只是一个不带截止时间的待办记录，不启用提醒，也不启用循环。

2. 为什么错过任务不在历史里？

因为 已错过 仍然是活动态。设计上是希望你还能处理它，而不是让它在历史里默默躺尸。

3. 为什么批量导入第一条必须带日期？

因为后续省略日期的条目会继承上一条的日期。没有第一条日期，就无法推断后面的归属日。

4. 为什么日程提醒既能选系统通知提示音，又能选播放通道？

声音文件来源和播放通道是两回事。系统通知提示音只是音色来源；真正播放时可以按设置选择闹钟、无障碍辅助、通知或媒体通道，并叠加 PaykiTodo 内部音量。

5. 为什么循环任务编辑前会多一步范围选择？

因为不先确认你是只改当前一条，还是要影响未来整个系列，系统没法安全地修改循环数据。

6. 如果我换手机，电脑现在能直接看到我手机上的待办吗？

不能直接看到。当前版本还是本地单机模式，不会自动把手机数据库同步到电脑。你需要主动导出 JSON，或者把自动备份目录放到一个可同步的位置。

7. 为什么提醒到点后最好不要只盯着通知栏？

因为真正强提醒的主链路是全屏界面、前台服务和诊断兜底链路的组合。通知栏只是其中一层，不是全部。
