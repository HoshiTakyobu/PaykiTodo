## Part 1：规划台快捷栏新增「公告」按钮

### 文件

- `app/src/main/java/com/example/todoalarm/ui/PlanningDeskPanel.kt`

### 现状

`PlanningShortcutSpec` 列表在 line 680 附近，10 个按钮：任务/子任务/缩进/减少缩进/DDL/日程/提醒/分组/今日/明日。没有公告。

`PlanningShortcutAction` sealed interface 定义在 line 1767：现有 `Insert(token)`、`InsertSection(heading)`、`TaskLine`、`SubtaskLine`、`Indent`、`Outdent`。

### 设计

在「明日」后面新增「公告」按钮：

```kotlin
PlanningShortcutSpec(
    "公告",
    Icons.Rounded.Campaign,
    PlanningShortcutAction.InsertAnnouncement,
    "在新行插入 #公告 占位，填上日期范围和正文后会显示在每日看板顶部和桌面小组件"
)
```

新增 `PlanningShortcutAction.InsertAnnouncement: PlanningShortcutAction` 单例。

### 行为

执行 `InsertAnnouncement` 时（参考现有 `InsertSection` 实现，line ~1900 区域）：

1. 当前光标所在位置如果不是行首，先插入换行
2. 插入字符串 `#公告 `（带尾部空格）
3. 光标停在 `#公告 ` 之后，等用户继续输入

实现上接近 `InsertSection`，但插入的是行内 token（不是标题），且不强制后面再加一个空行。可以单独实现一个 `insertAnnouncementAtNewLine(value: TextFieldValue): TextFieldValue` 函数。

### 视觉

`Icons.Rounded.Campaign` 是喇叭图标，与公告语义一致。`DashboardChrome.kt` 的 `AnnouncementBanner` 已经用过这个图标，视觉统一。

### 验证

- 点击「公告」按钮，编辑器在新行插入 `#公告 `，光标在尾部
- 用户接着输入 `5.20-5.25 测试公告`，整行变成 `#公告 5.20-5.25 测试公告`
- 切换到 Markdown 预览，看到橙色"全局公告"卡片
- 回到看板，公告横幅显示

## Part 2：AI 日报使用指引

### 入口设计

用户当前在「设置 -> AI 调用配置 -> AI 日报 / 周报」卡片里只看到一段简短说明：

> 日报会在每天设定时间生成并写入规划台「AI 日报」；周报会在每周日设定时间写入「AI 周报」。AI 配置不可用时会用本地模板兜底。若未开启「精确闹钟」权限，系统可能延后生成。

这段过于浓缩。要新增一个「了解 AI 日报」按钮（或问号图标），点击弹出 ModalBottomSheet 详细讲解。

### 文件

- `app/src/main/java/com/example/todoalarm/ui/SettingsPanel.kt` — 在 PrefCard("AI 日报 / 周报") 内部，"立即生成一次日报" 按钮上方加一个 `OutlinedButton(onClick = { showHelpSheet = true }) { Text("了解 AI 日报") }`
- 新建 ModalBottomSheet 显示详细指引

### Sheet 内容（按以下结构组织成多个 Card）

**Card 1：什么是 AI 日报**
- AI 日报会汇总你今天完成的待办、错过的待办、今日和明日日程、明日 DDL、今日专注分钟数
- 调用你已配置的 AI 源生成 1-3 段中文总结，并给出「明天先做什么」的建议
- 周报在每周日生成，覆盖本周累计数据

**Card 2：使用前提**
- 至少配置一个可用的 AI 源（DeepSeek / Qwen / OpenAI 兼容），并通过「测试连接」
- 如果所有 AI 源都不可用，会自动用本地模板生成简短报告，不会生成空内容
- Android 12+ 需要授予精确闹钟权限，否则系统可能延后几小时甚至跳过生成

**Card 3：启用步骤**
1. 在本卡片打开「日报」开关，设置生成时间（默认 22:00）
2. 可选：打开「周报」开关
3. 首次使用建议立即点底部的「立即生成一次日报」验证
4. 报告生成后会有通知栏推送，点通知直达对应文档

**Card 4：报告在哪看**
- 通知栏 → 点击「AI 日报已生成」通知 → 自动打开规划台「AI 日报」文档
- 抽屉 → 规划台 → 选择文档 → 「AI 日报」或「AI 周报」
- 报告文档顶部会有紫色提示条，说明这是自动生成的复盘记录

**Card 5：常见问题**

- **没收到报告？**
  - 检查 AI 源是否能通过「测试连接」
  - Android 12+ 系统设置里授予 PaykiTodo「闹钟和提醒」权限
  - 检查通知权限是否开启
  - 如果手机在深度休眠，可能延后；点「立即生成一次日报」可以手动触发
- **报告很简短只有几行？**
  - AI 不可用时走本地模板，只列出"今天完成 N 条"+"明天 DDL"，没有 AI 风格的总结建议
  - 检查 AI 配置和网络
- **AI 写的内容不准？**
  - 报告基于当天数据库快照生成，22:00 之后完成的任务不会进入当天日报（会进次日）
  - AI 偶尔会幻觉，如果发现严重错误可以反馈给开发者改进 prompt

### Sheet 视觉

参考现有 `PlanningHelpSheet`（PlanningDeskPanel.kt 内部）的卡片风格：每个 Card 用 `Surface` 圆角 + tonalElevation，标题加粗，正文 bullet。
