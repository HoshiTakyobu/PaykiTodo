PaykiTodo 使用说明

待办 强提醒 日历 批量导入 模板与备份

这份说明的目标不是告诉你“软件大概能做什么”，而是告诉你“某个按钮点进去到底怎么用、会发生什么、遇到什么边界情况是正常的”。如果你对软件中的某个操作有疑惑，应该能在左侧目录里找到对应入口。

推荐阅读顺序：第一次上手 → 首页与待办列表 → 全屏提醒界面 → 日历界面总览 → 批量导入语法 → 提醒权限、诊断与提示音。

待办系统

适合做需要强提醒、需要全屏弹出、需要完成/取消/延后的事情。

日历系统

适合课程表、值班表、全天日程、跨天安排和时间轴规划。

诊断与备份

提醒链路测试、崩溃日志、导出 JSON、自动备份都在软件内。

当前版本最重要的交互原则

  • 活动态只有三种：已错过、今日待办、计划中。其中 已错过 仍然是活动态，不算历史态。
  • 真正进入历史记录的只有两种状态：已完成 和 已取消。
  • 无 DDL 的待办可以存在，但它不会启用提醒，也不会启用循环。
  • 循环任务/循环日程在编辑、取消、删除时，常常会先要求你选择影响范围。
  • 提醒到点后的主目标是让你明确操作，不再保留“忽略一下先关掉”的入口。

第一次上手

如果你刚装好软件，建议先按下面的顺序做一轮完整配置。这样后续测试提醒时不会被系统权限或后台限制误伤。

  1. 进入 设置 → 提醒权限，把通知、精确闹钟、全屏提醒、免打扰穿透、忽略电池优化、辅助功能提醒都检查一遍。
  2. 进入 设置 → 提醒链路诊断，先跑一次短延迟测试，确认通知、前台服务、全屏页、锁屏显示链路至少能通。
  3. 进入 设置 → 提示音，如果你不想用内置音，改成系统通知提示音。
  4. 回到首页，先建一条有 DDL、带提醒的普通待办，再建一条无 DDL 的随手任务，对比两种行为差异。
  5. 打开日历，先手动创建一个普通日程，再试一次批量导入，熟悉两种录入方式。
  6. 最后到 设置 → 数据与备份 里选一个备份目录，避免后续卸载应用导致本地数据直接丢失。
PaykiTodo 目前仍然是本地单机优先。卸载应用会清掉应用私有目录里的数据库、设置和语录缓存。要保数据，必须主动备份或导出。

输入框旁边的问号是什么

提醒时间、批量导入、自定义延后等输入框旁边会出现一个圆圈里的问号。点一下会弹出当前输入框的语法说明和示例，不需要你记住所有规则。

提醒时间统一写法

写法含义
5提前 5 分钟提醒。
5,15提前 5 分钟和 15 分钟分别提醒。编辑待办/日程时可多选。
16:30DDL 或日程开始当天的 16:30。
05-10 15:00当年 5 月 10 日 15:00。
2026-05-10 14:30完整日期时间。
如果任一提醒时间晚于 DDL / 日程开始时间，或新建时已经过去，输入框会标红，保存按钮会禁用。

抽屉菜单与导航

点左上角三道杠会打开抽屉菜单。抽屉不是全屏覆盖，而是一个左侧面板，主要用于切换模块和筛选分组。

顶部区域

  • 最上方显示软件图标和 PaykiTodo 名字。
  • 底部有显示模式切换按钮。点击后会弹出 浅色、深色、跟随系统 三种模式。
  • 如果当前是 跟随系统，图标会根据系统当前明暗显示为太阳或月亮。

菜单结构

入口作用
我的任务可展开。展开后先看到 全部任务，再看到所有分组筛选项。
日历进入时间轴/周视图/月视图/列表视图所在的日历模块。
历史记录查看已完成和已取消的任务。
分组管理新建、改名、改颜色、删除自定义分组。
设置目录式设置页。大部分调试与权限项都在这里。
关于版本号、内部版本、包名、项目信息和说明入口。

分组筛选是怎么生效的

  • 在 我的任务 下点击某个分组后，首页仍然保持 已错过 / 今日待办 / 计划中 这三个板块。
  • 变化的是内容过滤：只显示当前分组内的任务。
  • 点击 全部任务 则取消过滤。

首页与待办列表

首页是 PaykiTodo 的待办主战场。它保留语录、分区、预览卡片和直接勾选完成的能力。

语录卡片

顶部显示一句本地缓存语录。右侧刷新按钮可以切到下一条，退出后会记住你停留的位置。

已错过

显示第一天及以前已过 DDL 但未完成、未取消的任务。默认展开；如果没有错过任务则自动隐藏。

今日待办

显示今天应处理的任务。默认展开。

计划中

显示未来任务，以及无 DDL 任务。默认只有在今日待办为空时才会自动展开。

任务排序规则

  • 已错过：按时间从早到晚排。
  • 今日待办：按今天内的时间从近到远排。
  • 计划中：先未来有 DDL 的任务，再无 DDL 的任务。无 DDL 会排在计划中末尾。

任务卡片怎么看

  • 左侧竖条颜色来自任务所属分组。
  • 标题前的小标签显示分组名。
  • 有备注时，卡片会直接展示备注摘要。
  • 有 DDL 时，会显示 ⏰ DDL。
  • 循环任务会额外显示 循环任务 字样。
  • 错过任务的标题、备注、DDL 会偏红，便于快速识别。

快速完成

点卡片最左侧复选框即可完成任务。完成时会有划线和烟花过渡动画，然后任务会移动到历史记录。

创建与编辑待办

首页右下角的加号默认用于新增待办。编辑时则从任务卡片预览窗里的 修改 按钮进入。

普通待办字段

字段说明
标题必填。卡片和提醒页都会优先展示标题。
备注可为空。首页卡片最多展示摘要，完整内容在预览和提醒页查看。
所属分组决定颜色、标签和筛选归属。
设置 DDL打开后才有截止日期和提醒逻辑；关闭则变成无 DDL 任务。
启用提醒只有有 DDL 的活动任务才能启用。
提醒时间使用输入框填写。可写提前分钟、当天时刻、当年日期时间或完整日期时间；输入框旁边的圆形问号会显示示例。
响铃 / 震动两者独立开关，按你的选择触发。

无 DDL 任务

  • 把 设置 DDL 关掉即可创建无截止日期任务。
  • 无 DDL 任务会排在 计划中 末尾。
  • 无 DDL 任务不会启用提醒，也不会启用循环。

循环任务开关

任务层面不额外区分“任务类型”，而是通过 循环任务 开关决定当前待办是否按循环方式生成多个实例。

目前支持的循环规则

  • 每天
  • 每周的周几，支持多选
  • 每月的第几个星期几
  • 每月的 D 日
  • 每年的 M 月 D 日

循环截止日期

  • 循环任务必须设置截止日期。
  • 默认值是首个任务日期后的 90 天。
  • 可以手动修改。

每月 D 日的特殊规则

  • 如果你选的是 29、30、31 这类日期，而某个月没有这一天，则自动落到该月最后一天。
  • 例如选择每月 31 日，到了 4 月会生成在 4 月 30 日。
  • 编辑器下方会显示一行红色小字提示这个规则。

预览循环生成

点 预览循环生成 可以先看这个循环配置大概会生成多少条实例，以及前几条的实际落点。建议你在课表、吃药、值班这种重要循环项上先预览再保存。

编辑循环任务时的范围选择

如果你修改的是仍处于活动态的循环任务，系统会先弹出“选择修改范围”。

范围含义
当前事件只改当前这一条实例。
当前事件和此后所有事件从当前实例开始，当前和未来一起改。
所有事件整个系列全部改。

待办预览、完成、取消、恢复

点击任务卡片主体会打开预览窗。很多不适合直接暴露在列表里的按钮，现在都放在预览窗里了。

活动任务预览窗

  • 显示分组、标题、备注、创建时间、DDL。
  • 底部按钮一般是 取消、关闭、修改。
  • 如果是循环任务，点 取消 或 修改 前通常会先要求选择范围。

取消任务

  • 取消后任务会停止提醒，并进入历史记录。
  • 一次性任务通常直接取消当前这条。
  • 循环任务会先要求选择作用范围。

历史任务预览窗

  • 历史列表中的卡片只显示分组、标题、备注，保持界面简洁。
  • 更多细节例如创建时间、DDL、完成/取消时间，放在预览窗里查看。
  • 历史预览窗里可以看到 恢复 和 修改。

恢复任务

历史记录中的任务可以通过 恢复 回到活动态。恢复后是否继续参与提醒，要看当前任务本身的时间条件和状态，而不是无条件重新响铃。

错过任务是怎么产生的

  • 任务过了 DDL 后，如果在短缓冲窗口内仍未完成、未取消、未改期，它会进入 已错过。
  • 进入 已错过 后仍然是活动态，不会直接丢去历史。
  • 你仍然可以点开它进行修改、完成或取消。

电脑同步

从 1.6.0 开始，PaykiTodo 内置了一个局域网网页控制台。手机和电脑在同一局域网时，电脑浏览器可以直接连接手机，查看并修改手机里正在使用的同一份待办和日程数据。

这不是单独的 Windows 客户端，而是手机端开出一个网页控制台。优点是不用再装一个桌面程序，电脑打开浏览器就能用。

入口在哪里

  1. 进入 设置 → 电脑同步。
  2. 打开 启用电脑同步 开关。
  3. 记下页面里显示的 访问密钥。
  4. 记下页面里显示的局域网地址，例如 http://192.168.1.23:18765。
  5. 在电脑浏览器里打开这个地址。
  6. 在网页左侧输入访问密钥，点 连接手机。

当前已经支持什么

  • 查看手机中的待办列表。
  • 查看手机中的日程列表。
  • 在电脑端直接新增普通待办。
  • 在电脑端直接新增普通日程。
  • 把待办标记为已完成。
  • 取消待办。
  • 删除日程。
  • 所有改动直接写入手机本地数据库，并复用手机端现有的提醒调度逻辑。

当前限制

  • 第一版优先保证同局域网直连，不是公网云同步。
  • 电脑端当前是轻量控制台，主要强调“能快速录入和同步”，不是完整复刻手机端全部 UI。
  • 网页端目前先支持基础新增与处理操作，复杂循环编辑、模板操作和批量导入暂时仍建议在手机端做。
  • 如果手机没有连 Wi‑Fi，设置页里可能看不到可访问的 IPv4 地址。

它是怎么同步到手机的

网页控制台操作的不是一份“另外的副本”，而是手机应用当前正在使用的 Room 数据库。也就是说，电脑上新增的待办和日程，会直接出现在手机端；手机端完成或删除的内容，下次网页刷新时也会同步看到。

安全性说明

  • 网页接口要求你输入 访问密钥 才能操作。
  • 如果怀疑密钥泄露，可以在设置页里点 重新生成访问密钥。
  • 建议只在你信任的局域网里开启这项功能。

遇到连不上时怎么排查

现象排查方向
电脑打不开地址先确认手机和电脑是否连的是同一个 Wi‑Fi；再确认手机设置页显示的地址有没有抄错。
网页能打开但提示未授权说明地址没问题，但访问密钥不对。重新输入，或者在手机里重置密钥。
开关打开后还是看不到地址通常是手机当前没有拿到可用的局域网 IPv4 地址，例如未连 Wi‑Fi。
手机重启后无法连接重新打开设置页确认电脑同步是否仍为开启状态；如果开启，服务会自动恢复。

分组管理

分组既是视觉颜色来源，也是筛选维度。默认分组和自定义分组都在这里管理。

你能做什么

  • 新建分组
  • 修改分组名称
  • 修改分组颜色
  • 删除自定义分组

不能直接删除的情况

  • 默认分组不能直接删。
  • 仍被活动任务使用的分组不能直接删，需要先迁移或清空相关任务。

历史记录

历史记录只保留真正结束的事项，不承接仍待你处理的错过任务。

会进入历史记录的状态不会进入历史记录的状态
已完成已错过
已取消今日待办

列表为什么这么简化

历史记录列表现在只直接展示分组名、标题、备注，是为了避免一条记录上同时塞进三个时间、两个按钮导致界面太乱。详细信息放在点击后的预览窗里查看。

全屏提醒界面

当待办或日程到点后，如果触发的是全屏提醒模式，你会看到一个独立的全屏界面，而不是仅靠通知栏轻飘飘地提醒一下。

这个界面的目标

  • 尽量在锁屏、后台、解锁后都把界面拉到前台。
  • 不允许简单按返回键就关掉提醒页。
  • 要求你明确作出一个动作，而不是继续拖延。

待办提醒会出现哪些按钮

  • 我已完成
  • 延后 5 分钟
  • 取消任务
  • 延后 X 分钟，X 来自设置里的默认延后时长
  • 自定义延后，可输入分钟数，也可输入未来具体时刻，例如 16:30。

日程提醒和待办提醒的区别

  • 日程提醒的主按钮是 我知道了。
  • 日程提醒不需要“继续延后再等你做”的那套逻辑，所以不会和待办完全一样。
  • 日程会尽量把 日期 和 时间段 直观显示出来。

延后时间的对齐规则

延后不是简单的“当前时间 + N 分钟 + 保留秒数”，而是按分钟对齐。也就是你在任意秒数点下延后后，新的提醒时刻会把秒和毫秒清零。

例如现在是 11:05:38，你点 延后 1 分钟，预期时刻应该是 11:06:00，而不是 11:06:38。

循环任务在提醒页取消时的范围选择

如果当前提醒对应的是循环任务，按 取消任务 后会先弹出范围选择：

  • 当前事件
  • 当前事件和此后所有事件
  • 所有事件

提醒权限、诊断与提示音

设置页目录现在怎么看

设置首页现在是目录式的，不再把所有内容堆成一个超长滚动页。你会看到三个大组：

  • 提醒与权限
  • 日历与声音
  • 数据与帮助

提醒权限

进入 设置 → 提醒权限 后，会看到每项权限的当前状态，以及右侧的 去设置 按钮。

权限用途
通知权限允许发通知和前台服务通知。
精确闹钟让系统在更准确的时间派发提醒。
全屏提醒允许使用系统支持的全屏提醒链路。
免打扰穿透在你的系统支持且已授权时，减少提醒被静音策略压掉的概率。
忽略电池优化减少后台派发被 ROM 杀掉的风险。
辅助功能提醒作为额外兜底链路，提高回到提醒页的机会。

提醒链路诊断

  • 进入 设置 → 提醒链路诊断。
  • 点 提醒链路测试，输入多少秒后提醒。
  • 测试任务会短延迟创建，用来快速验证通知、前台服务、全屏页和无障碍兜底链路。
  • 下方最近诊断记录会显示每个阶段的状态，便于定位到底卡在哪一步。
  • 点 清空诊断 可以重置这一页的链路记录。

日历与提醒

这里主要放影响全局默认行为的选项。

设置项作用
默认延后时长影响全屏待办提醒界面里的默认延后按钮。
日历周起始日影响周视图和按周计算时，一周是从周一还是周日开始。
日历提醒默认方式新建日历日程时默认是通知栏提醒还是全屏界面提醒。

提示音

  • 提醒真正播放时仍然走闹钟音量通道，这是为了保证强提醒的可听性。
  • 但你选择的声音来源，可以改成系统里的通知提示音。
  • 使用内置提醒音：恢复软件自带音效。
  • 选择系统提示音：打开系统铃声选择器，读取通知提示音列表。
也就是说：音量走闹钟，但音色来源可选通知提示音。这两件事不是一回事。

日历界面总览

日历模块是用来承接课程表、值班表、时间轴规划和跨天/全天安排的，不只是把待办换个壳。

顶部一排控件分别是什么

控件作用

时间轴界面的总体结构

  • 最左侧固定为时间轴。
  • 上方先显示日期头。
  • 日期头下方是全天栏。
  • 最下面的大区域才是按小时划分的时间网格。
  • 红线表示当前时间；今天的日期会高亮。

四种视图

1. 时间轴

  • 最适合查看一天内的排布、持续时长和时间冲突。
  • 全天日程显示在时间网格上方的全天栏。
  • 跨天日程会按跨越的日期拆分到对应天中显示，不会只落在第一天。
  • 空白时间段点击一次会出现预添加块，再点一次才真正进入创建面板。
  • 预添加块会自动吸附到最接近的 15 分钟刻度，并默认持续 30 分钟左右的初始长度。

2. 周视图

  • 适合快速扫一周内的安排。
  • 周视图的“这一周”是受 设置 → 日历与提醒 → 日历周起始日 控制的。
  • 如果你习惯从周日开始看一周，可以在那里切换。

3. 月视图

  • 适合看某个月内有哪些安排，但不强调小时级布局。
  • 会按当前中心月份筛出该月的日程。

4. 列表

  • 按时间顺序直接列出全部日程。
  • 适合做总检查、找一条很久之后的安排，或在时间轴太密的时候快速扫一遍。

创建与编辑日程

日程编辑器比待办多了地点、颜色、全天、开始/结束时间和提醒派发方式。

基本字段

字段说明
日程标题主显示内容。时间轴卡片优先保标题，空间足够时再展示地点。
地点显示在日程卡片副标题和提醒页地点行。
备注详情页和提醒页会显示。
全天日程开启后显示在全天栏，不占小时网格。
开始时间 / 结束时间普通日程按时间显示；跨天时结束日期可晚于开始日期。
颜色影响日程块主色。

提醒设置

  • 先打开 提醒 开关。
  • 提醒时间 使用输入框填写，输入框旁边的圆形问号会显示写法说明。
  • 可写 5,15 这类提前分钟，也可写 16:30、05-10 15:00、2026-05-10 14:30 这类具体时刻。
  • 提醒方式 可选 通知栏提醒 或 全屏界面提醒。
  • 响铃 与 震动 可分开控制。

全天日程的提醒基准

全天日程没有具体开始时分时，系统默认按当天 09:00 作为提醒计算锚点。因此你设“提前 1 天”时，实际会相对这一天的 09:00 来推算。

更多按钮、周模板、学期生成

日历右上角的 … 不是杂项垃圾桶，它主要承接高效率录入功能。

批量导入

适合一次性导入课程表、值班表、活动列表。详细语法放在下一个章节单独说明。

周模板

  • 用于管理已经保存的周模板或学期模板。
  • 你可以把模板复制到某一周。
  • 也可以基于模板直接生成整学期循环日程。
  • 模板不需要你每次从零重新录课表。

保存本周模板

  • 把当前可见周内的日程保存为模板。
  • 保存时可命名，并标记模板类型，例如课程表模板、值班周模板、学期模板。

批量导入语法

如果你经常拿到一整学期课表、某周值班安排，或者想快速规划某天的时间，批量导入会比手动点格子高效很多。

入口在哪里

  • 日历页顶部点独立的 批量 按钮。
  • 进入导入窗后，可直接粘贴文本，也可先点 载入示例
  • 输入框旁边的圆形问号会显示当前语法提示。

待办批量添加

每日看板和我的任务页有 批量添加待办。它是为了快速手输，所以语法比日程更轻。

待办格式
DDL时间,任务名称,提醒时间
示例
2026-05-12 18:00,写报告,5
05-13 09:30,给老师发消息,09:00
无DDL,整理 Obsidian 待办
  • 提醒时间可省略。
  • 为了手输简单，一行待办只支持一个提醒时间。
  • 默认使用默认分组、默认响铃和默认震动设置。
  • 无DDL 的待办不能设置提醒。

最小可用语法

示例
2026-04-27: 10:20-11:55, 辅导员助理值班, @MB-B1-412; 12:30-14:00, 午休
  • 第一条必须带日期。
  • 同一天后续条目可以省略日期。
  • 条目之间可以用英文分号 ;，也可以直接换行。
  • 最少需要 时间段, 标题 这两个字段。

字段顺序

推荐顺序如下：

推荐顺序
[日期:] 时间段, 标题, 地点, 循环规则, 循环截止日期, 扩展字段

时间段写法

写法含义
10:20-11:55普通同日时间段
22:30-次日01:00跨天日程
全天全天日程

地点前面的 @ 现在怎么处理

@ 不再被当作语法标记，而是普通字符串的一部分。也就是说，@品学楼C506 会原样当作地点文字保存下来。如果你想让地点前面带 @，直接写就行。

如果内容里有逗号或分号怎么办

请用双引号包起来。

示例
2026-03-02: 19:30-21:55, "【课】二胡演奏基础", "@品学楼,C506"

循环规则写法

写法含义
Daily每天
Weekly按当前日期对应星期每周循环
Weekly[Mon,Wed,Fri]每周一、三、五循环
MonthlyDay每月同一日
MonthlyNthWeekday每月第 N 个星期 X
Yearly每年同月同日
示例
2026-03-02: 19:30-21:55, 【课】二胡演奏基础, @品学楼C506, Weekly, 2026-05-12
09:00-10:05, 【课】辅导员助理值班, @MB-B1-412, Weekly, 2026-06-15

提醒扩展字段

可选扩展字段用 键=值 的形式书写。

字段示例作用
NoteNote="刷题 2 套"备注
RemindRemind=5 / Remind=16:30提醒时间。支持提前分钟、当天时刻、当年日期时间、完整日期时间；off 表示不提醒。
ModeMode=Notification通知栏提醒或全屏提醒
RingRing=On是否响铃
VibrateVibrate=Off是否震动
ColorColor=#4E87E1日程主色
GroupGroup=课程导入时写入分组名，供后续映射

默认规则

  • 不写循环规则：按单次日程处理。
  • 写了循环规则但不写截止日期：默认截止到起始日期后 90 天。
  • 批量导入默认提醒是：提前 5 分钟 + 通知栏提醒 + 响铃开启 + 震动开启。
  • 日程批量导入的 Remind= 支持和编辑界面相同的提醒时间写法。

导入前建议先做什么

  1. 先点 解析预览，确认成功条数和报错条数。
  2. 检查预览卡片里的时间、地点、循环标签和提醒方式。
  3. 确认无误后再点 一键导入。

备份与恢复

PaykiTodo 当前不是云同步产品，所以备份是你避免数据丢失的第一道保险。

入口

进入 设置 → 数据与备份。

你会看到哪些按钮

按钮作用
选择备份目录选择自动备份写入的位置。
导出 JSON手动导出当前所有数据。
导入 JSON从以前导出的 JSON 恢复数据。
自动备份 开关开启后，任务、模板或分组发生变动时会尝试写入备份目录。

备份里包含什么

  • 任务实例
  • 循环模板
  • 分组
  • 提醒链路诊断日志
  • 周模板 / 学期模板
  • 业务设置，例如主题模式、周起始日、默认延后时长、提醒音来源等

导入会怎样

导入目前更偏“整库恢复”思路，而不是复杂合并。也就是说，如果你准备从一个旧 JSON 恢复，请先确认现有数据是否已经另外导出保存。

崩溃日志与常见排查

软件现在只会在检测到真正的上次异常退出时，才提示你去设置页查看崩溃日志。

怎么看崩溃日志

  1. 进入 设置 → 崩溃日志。
  2. 如果有新日志，可以点 查看日志。
  3. 点 复制 就能把文本拷出来给我分析。
  4. 确认没用了以后，可以点 清空日志。

提醒不稳定时先看哪里

  • 先看 设置 → 提醒权限 是否齐全。
  • 再看 设置 → 提醒链路诊断 的最近记录。
  • 如果是直接闪退，再去看 崩溃日志。

常见问题

1. 为什么我创建了无 DDL 任务，却没有提醒？

这是预期行为。无 DDL 任务只是一个不带截止时间的待办记录，不启用提醒，也不启用循环。

2. 为什么错过任务不在历史里？

因为 已错过 仍然是活动态。设计上是希望你还能处理它，而不是让它在历史里默默躺尸。

3. 为什么批量导入第一条必须带日期？

因为后续省略日期的条目会继承上一条的日期。没有第一条日期，就无法推断后面的归属日。

4. 为什么日程提醒既能选系统通知提示音，又说走闹钟音量？

声音文件来源和音量通道是两回事。PaykiTodo 当前保持“音量走闹钟通道”，同时允许你从系统通知提示音列表中挑一个更适合作为提醒声的音色。

5. 为什么循环任务编辑前会多一步范围选择？

因为不先确认你是只改当前一条，还是要影响未来整个系列，系统没法安全地修改循环数据。

6. 如果我换手机，电脑现在能直接看到我手机上的待办吗？

不能直接看到。当前版本还是本地单机模式，不会自动把手机数据库同步到电脑。你需要主动导出 JSON，或者把自动备份目录放到一个可同步的位置。

7. 为什么提醒到点后最好不要只盯着通知栏？

因为真正强提醒的主链路是全屏界面、前台服务和诊断兜底链路的组合。通知栏只是其中一层，不是全部。
