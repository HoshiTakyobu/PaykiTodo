# PaykiTodo 规划台当前用法与范例

适用版本：v1.14.0 起。

规划台现在是一张带识别能力的备忘录：先自由写，再点击“识别计划”，最后在预览里确认并导入。当前主流程不要求学习 Markdown 标题、井号标签或大纲节点。

## 1. 最常用写法

```text
10:00-12:00 写论文 @图书馆3楼
12:00-13:00 吃饭
任务M ddl 15:00
明天 16:30 交材料
整理保研材料
打印成绩单
```

识别结果：

- 有起止时间的行会进入日程候选。
- 写了 `ddl`、`截止`、`前`、明确日期时间的行会进入待办候选。
- 没写 DDL 的普通行会作为无 DDL 待办候选，导入后按今日待办展示。
- 所有结果都只进入预览，不会直接写入正式待办或日程。

## 2. 日程示例

```text
10:00-12:00 写论文 @图书馆3楼
5.28 14:00-16:00 小组讨论
明天 19:30-21:00 整理保研材料
周五 14:00-16:00 写论文
10:00-12:00, 【课程】习思想, "@主楼B1-412"
```

规则：

- 未写日期的日程默认按今天预览，并在预览中提示确认。
- `@图书馆3楼`、`地点：主楼B1-412`、`"@主楼B1-412"` 会识别为地点；`@` 是用户输入的一部分，不会自动重复添加。
- 创建日程时默认不会同步创建待办；如需关联待办，在预览或正式日程编辑界面里手动开启。

## 3. 待办与 DDL 示例

```text
任务M ddl 15:00
交材料 ddl 5.29 23:59
5.29 把入党资料交到芳姐那里
明天 16:30 交材料
周五 写论文 ddl 23:59
16:30之前发邮件
5点前交作业
```

规则：

- 只写日期不写时间时，默认当天 23:59。
- 只写时间不写日期时，默认今天对应时间；如果已经过期，预览会阻止直接导入并提示修改。
- 文档标题或文档日期不再作为隐藏日期上下文。正文里写了哪一天，就按哪一天识别。

## 4. 无 DDL 待办

```text
整理保研材料
打印成绩单
问老师课程论文格式
```

这些内容会识别为无 DDL 待办候选。它们适合承接“先把脑子里的事倒出来”的工作流，不强迫你一开始就把所有时间定死。

## 5. 大任务和子任务

第一阶段仍以文本和预览为主，不做复杂拖拽树。你可以直接按普通备忘录方式写：

```text
保研材料准备
  整理成绩单
  整理获奖证明
  写个人陈述初稿 ddl 5.24
```

解析器会尽量保留父任务备注；如果识别不准，在预览里修改即可。

## 6. AI 和本地规则

- 如果设置里启用了可用 AI 源，规划台会优先让 AI 识别自由文本。
- AI 失败、超时或未配置时，会自动回到本地规则。
- 明确的时间段和 DDL 本地规则会直接给出预览；普通无 DDL 文本属于低置信兜底，AI 可继续接管。

## 7. 导入后的去重

导入后不会向原文追加可见标记。软件通过内部映射记录已导入行，再次识别同一行时会默认跳过，保持备忘录正文干净。
