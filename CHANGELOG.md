# Changelog

PaykiTodo 采用语义化倾向的应用版本号和递增 `versionCode`。本文件面向用户记录重要变化，不再保存每一次内部调试重建的完整流水账；完整历史可通过 Git commit、tag 和 `docs/archive/` 追溯。

## v1.13.14

发布日期：2026-05-24（本地调试构建，尚未创建 GitHub Release）

### 修复

- 电脑同步服务改为持续检测授权桌面端心跳：电脑端未输入访问密钥，或曾连接后断开超过 5 分钟，手机端会自动关闭电脑同步开关。
- 电脑同步前台通知区分“等待连接”和“已连接”，自动关闭时会停止前台服务并移除通知。
- 电脑端 Web 连接成功后每 60 秒发送一次授权心跳，关闭网页、断网或电脑休眠后不再维持手机端同步服务。
- 手机端状态读取不再裸启动局域网 HTTP server；开关开启但服务未运行时会优先启动前台服务，避免出现可访问但通知栏没有同步服务的状态。

### 构建

- `versionName = 1.13.14`
- `versionCode = 262`
- 数据库版本保持 `25`，不新增数据迁移。

## v1.13.13

发布日期：2026-05-24（本地调试构建，尚未创建 GitHub Release）

### 修复

- 电脑端 Web 待办编辑器的标题输入改为多行输入框，支持输入和保存手机端已有的换行标题。
- 电脑端待办/日程列表与详情预览保留标题换行显示，不再把多行标题压成一行。
- 电脑端编辑循环待办或循环日程时新增“修改范围”，支持仅当前、当前及之后、整个循环系列。
- 电脑端把循环待办改为“不重复”时，默认按“当前及之后”处理，避免未来已生成的循环实例继续出现。
- 桌面同步后端 `PUT /api/todos/{id}` 与 `PUT /api/events/{id}` 接收循环修改范围，并在删除/取消接口中支持范围参数。

### 构建

- `versionName = 1.13.13`
- `versionCode = 261`
- 数据库版本保持 `25`，不新增数据迁移。

## 版本记录规则

- 面向用户的功能、行为、数据结构和发布信息写入本文件。
- 单纯内部调试、会话交接和未发布实验细节写入 `docs/current/` 或 Git commit，不在本文件堆积。
- 如果数据库版本变化，必须在对应版本明确写出迁移字段和兼容说明。
- 如果 release 已发布，记录 GitHub Release 链接和 APK 元数据。
