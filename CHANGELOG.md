# Changelog

## v1.7.17

- 修正规划台手机端顶部操作区过高的问题：操作区改为固定 56dp 的工具条，不再占用大块空白
- 操作区只保留预览 / 识别 / 文档列表 / 更多操作，文档标题与说明不再挤在同一行，避免窄屏下布局异常
- 自动保存状态保留为短文本提示，平时不占用主要操作空间
- 版本号升级到 `1.7.17` / `versionCode 174`

## v1.7.16

- 规划台手机端操作区从三行压缩为一行紧凑布局：标题 + 预览/识别按钮 + 溢出菜单，编辑区可视面积增加约 40%
- 次要操作（新建/重命名/使用说明/归档/删除）收入溢出菜单，移除手动保存按钮（依赖已有的 2 秒自动保存）
- 快捷栏从编辑器下方移到上方，减少被输入法遮挡的概率
- 暗色模式 onSurfaceVariant 对比度从 4.2:1 提升到 5.3:1，辅助文字在暗色背景下更清晰
- 电脑端 Web 控制台内容区增加 max-width: 1200px 约束，宽屏下不再过度拉伸
- 版本号升级到 `1.7.16` / `versionCode 173`

## v1.7.15

- 修正规划台解析优先级：一行同时包含自然时间段和显式 `#ddl` 时，按待办处理，避免用户明确写 DDL 却被识别为日程
- 日期上下文标题支持更自然的紧凑写法，例如 `# 周五计划`、`# 明天安排`、`# 5/28周末计划`；同时继续避免 `# 我的明天计划`、`# 后天的事` 这类描述标题被误判为日期
- 补充规划台解析器单元测试，覆盖显式 DDL 优先级和紧凑日期标题边界
- 版本号升级到 `1.7.15` / `versionCode 172`

## v1.7.14

- 规划台手机端新增 2 秒防抖自动保存，并在切换规划文档前先保存当前未保存内容，降低编辑后切换或退出导致正文丢失的风险
- 修复手机端快捷栏 AssistChip 双重点击监听，避免一次点击插入两次任务、标签或标题
- 修复规划台导入成功后 `#imported` 只更新界面、不立即持久化的问题，避免下次打开同一行重复导入
- 识别预览的导入按钮现在要求至少选中一条有效候选；新增“全选可导入项 / 全不选”，预览 Sheet 和文档列表改为按屏幕高度自适应
- 预览模式点击任务或文本可回到对应原文行；在文档中间按回车也会续写 Markdown 任务行；预览编辑 DDL / 起止时间补齐自然日期写法
- 电脑端 Web 规划台新增编辑器自动保存、切换文档前保存、`Ctrl+S` 保存、`Ctrl+Enter` 识别、导入前空选中拦截和预览全选控制
- 版本号升级到 `1.7.14` / `versionCode 171`

## v1.7.13

- 修正规划台自然日程标题污染：行内写前导日期时，标题不再包含日期和时间段，例如 `5.28 14:00-16:00 小组讨论` 会识别为标题 `小组讨论`
- 收紧日期标题上下文：普通标题会清空旧日期上下文，`# 我的明天计划` 不再被误判为明天，`# 5/28 周末计划` 这类日期加描述标题可以提供日期上下文
- 自然日程不再要求时间段必须在行首，`复习 14:00-16:00` 也会识别为日程；顶层日程也能作为缩进子任务的父级备注来源
- 补齐斜杠日期、常见全角分隔符、中文 AM/PM 和全角时间段符号的解析与测试覆盖
- 未实现的 `#today`、`#tomorrow`、`#important`、`#project` 不再被静默从标题中剥掉，并在 Wiki / 规划台设计文档中明确为暂不支持的语义标签
- 版本号升级到 `1.7.13` / `versionCode 170`

## v1.7.12

- 更新内置说明中日程批量 `Remind=` 的示例，说明其支持编辑界面同款提醒语法，并提示批量字段内逗号的使用边界

- 更新输入帮助、Wiki、README 和 current 文档，明确“推荐自然写法”和当前仍保留的日程批量严格格式边界
