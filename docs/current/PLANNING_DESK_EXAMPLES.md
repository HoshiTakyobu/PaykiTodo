# PaykiTodo 规划台详细用法与范例

## 1. 规划台到底是干什么的

规划台不是正式待办列表，也不是正式日历。

它更像一个带识别能力的备忘录：

```text
先把脑子里的任务、安排、想法写下来
  -> 再点“识别”
  -> 在预览里修正标题、DDL、时间、提醒、分组
  -> 最后导入为正式待办或日程
```

所以规划台适合写：

- 还没整理好的任务；
- 大任务下面拆出来的小任务；
- 今天 / 明天 / 本周的大致安排；
- 之后要转成正式待办或日程的内容。

## 2. `# 今日计划`、`# 收集箱` 这种标题是干什么的

以 `#` 开头的行是 Markdown 标题。

标题主要有两个作用：

1. **让文档更好读**：把内容分区，例如“收集箱”“今日计划”“本周计划”。
2. **给下面的任务提供上下文**：如果标题里有“今天”“明天”或具体日期，下面没有写日期的日程/DDL 更容易按这个日期理解。

### 2.1 `# 收集箱`

收集箱适合放“先别管时间，先记下来”的内容。

例如：

```markdown
# 收集箱
- [ ] 想一下保研材料还缺什么
- [ ] 问老师课程论文格式
- [ ] 整理 PaykiTodo 后续想法
```

这类内容不一定马上有 DDL，也不一定马上导入。

### 2.2 `# 今日计划`

今日计划适合写今天要做的事情。

例如：

```markdown
# 今日计划
- [ ] 10:00-11:30 写课程论文 #group 课程
- [ ] 15:00-16:00 整理保研材料 #group 保研
- [ ] 晚上复习操作系统 #ddl 23:59 #remind 30,5
```

如果某行写了明确时间段，例如 `10:00-11:30 写课程论文`，识别时会优先按日程处理。

### 2.3 `# 明天`

明天标题适合写明天的安排。

```markdown
# 明天
- [ ] 09:00-10:30 背英语单词 #group 学习
- [ ] 20:00-21:30 完成实验报告 #ddl 21:30 #remind 15,5
```

### 2.4 `# 本周计划`

本周计划适合写还没精确排到某一天的任务。

```markdown
# 本周计划
- [ ] 完成课程论文初稿 #ddl 5.28 23:59 #group 课程
- [ ] 整理保研竞赛材料 #ddl 5.30 18:00 #group 保研
- [ ] 复习操作系统第 1-3 章 #group 学习
```

如果没有明确日期，系统不会凭空知道你要哪一天做。建议给重要任务写 `#ddl`。

## 3. 最常用写法

### 3.1 普通待办

```markdown
- [ ] 整理保研材料
```

### 3.2 带 DDL 的待办

```markdown
- [ ] 提交课程论文 #ddl 5.28 23:59
```

支持常见日期写法，例如：

```markdown
#ddl 5.28
#ddl 05.28 14:30
#ddl 5月28日 23:59
#ddl 2026-05-28 14:30
```

### 3.3 带提醒的待办

```markdown
- [ ] 交论文 #ddl 5.28 23:59 #remind 30,5
```

意思是提前 30 分钟和提前 5 分钟提醒。

### 3.4 分组

```markdown
- [ ] 写课程论文 #group 课程
- [ ] 整理保研材料 #group 保研
```

如果分组不存在，导入时会尽量映射到已有分组；识别不准时可以在预览里修改。

### 3.5 日程时间段

```markdown
10:00-12:30 写课程论文
明天 19:30-21:00 复习操作系统
5.28 14:00-16:00 小组讨论
```

有明确开始和结束时间的行，会优先识别为日程。

### 3.6 日程同时创建待办

规划台识别到日程后，预览里可以选择“同时创建待办”。

例如：

```markdown
10:00-12:30 写课程论文
```

可以生成：

- 一个 10:00-12:30 的日程；
- 一个 DDL 为 12:30 的待办。

## 4. 大任务和小任务

手机端可以点快捷栏里的“子任务”，也可以手写两个空格缩进。

```markdown
- [ ] 整理保研材料 #ddl 5.30 18:00 #group 保研
  - [ ] 打印成绩单
  - [ ] 整理获奖证书
  - [ ] 写个人陈述初稿
```

当前阶段，小任务会作为独立待办导入，并在备注里保留所属大任务的信息。

## 5. `#imported` 是什么

导入成功后，规划台会在原文行后面追加：

```markdown
#imported
```

例如：

```markdown
- [ ] 整理保研材料 #ddl 5.30 #imported
```

这表示这一行已经导入过正式待办或日程，主要用于避免重复导入。

你一般不需要手动写它。

## 6. 完整范例

```markdown
# 收集箱
- [ ] 想一下保研材料还缺什么
- [ ] 问老师课程论文格式
- [ ] 整理 PaykiTodo 后续想法

# 今日计划
- [ ] 10:00-11:30 写课程论文 #group 课程 #remind 5
- [ ] 15:00-16:00 整理保研材料 #group 保研
- [ ] 晚上复习操作系统 #ddl 23:59 #remind 30,5

# 明天
- [ ] 09:00-10:30 背英语单词 #group 学习
- [ ] 20:00-21:30 完成实验报告 #ddl 21:30 #remind 15,5

# 本周计划
- [ ] 完成课程论文初稿 #ddl 5.28 23:59 #group 课程
- [ ] 整理保研竞赛材料 #ddl 5.30 18:00 #group 保研
- [ ] 复习操作系统第 1-3 章 #group 学习
```

## 7. 推荐工作流

1. 打开规划台。
2. 在 `# 收集箱` 里先把脑子里的事情倒出来。
3. 把今天要做的移动到 `# 今日计划`。
4. 给重要任务补 `#ddl`、`#remind`、`#group`。
5. 点“识别”。
6. 在预览里修正不准确的条目。
7. 勾选要导入的内容。
8. 点“导入”。

## 8. 当前限制

- 规划台不是正式待办列表，正式提醒仍然依赖导入后的待办/日程。
- 手机端 Markdown 渲染是手动预览，不是默认启动界面。
- AI 自动拆解、拖拽排期、甘特图还没有实现。
- 如果识别不准，优先在预览页修改后再导入。
