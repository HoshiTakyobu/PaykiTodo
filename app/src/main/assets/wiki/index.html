PaykiTodo 使用说明

待办 强提醒 日历 批量导入 模板与备份

这份说明的目标不是告诉你“软件大概能做什么”，而是告诉你“某个按钮点进去到底怎么用、会发生什么、遇到什么边界情况是正常的”。如果你对软件中的某个操作有疑惑，应该能在左侧目录里找到对应入口。

推荐阅读顺序：第一次上手 → 首页与待办列表 → 全屏提醒界面 → 日历界面总览 → 批量导入语法 → 提醒权限、诊断与提示音。

待办系统

适合做需要强提醒、需要全屏弹出、需要完成/取消/延后的事情。

日历系统

适合课程表、值班表、全天日程、跨天安排和时间轴规划。

诊断与备份

提醒链路测试、崩溃日志、导出 JSON、自动备份都在软件内。

当前版本最重要的交互原则

  • 活动态只有三种：已错过、今日待办、计划中。其中 已错过 仍然是活动态，不算历史态。
  • 真正进入历史记录的只有两种状态：已完成 和 已取消。
  • 无 DDL 的待办可以存在，但它不会启用提醒，也不会启用循环。
  • 循环任务/循环日程在编辑、取消、删除时，常常会先要求你选择影响范围。
  • 提醒到点后的主目标是让你明确操作，不再保留“忽略一下先关掉”的入口。

第一次上手

如果你刚装好软件，建议先按下面的顺序做一轮完整配置。这样后续测试提醒时不会被系统权限或后台限制误伤。

  1. 进入 设置 → 提醒权限，把通知、精确闹钟、全屏提醒、免打扰穿透、忽略电池优化、辅助功能提醒都检查一遍。
  2. 进入 设置 → 提醒链路诊断，先跑一次短延迟测试，确认通知、前台服务、全屏页、锁屏显示链路至少能通。
  3. 进入 设置 → 提示音，可在 内置提醒音 和 系统通知提示音 之间切换。
  4. 回到首页，先建一条有 DDL、带提醒的普通待办，再建一条无 DDL 的随手任务，对比两种行为差异。
  5. 打开日历，先手动创建一个普通日程，再试一次批量导入，熟悉两种录入方式。
  6. 最后到 设置 → 数据与备份 里选一个备份目录，避免后续卸载应用导致本地数据直接丢失。
PaykiTodo 目前仍然是本地单机优先。卸载应用会清掉应用私有目录里的数据库、设置和语录缓存。要保数据，必须主动备份或导出。

输入框旁边的问号是什么

提醒时间、批量导入、自定义延后等输入框旁边会出现一个圆圈里的问号。点一下会弹出当前输入框的语法说明和示例，不需要你记住所有规则。

提醒时间统一写法

写法含义
5提前 5 分钟提醒。
5,15提前 5 分钟和 15 分钟分别提醒。编辑待办/日程时可多选。
16:30DDL 或日程开始当天的 16:30。
2:30 pmDDL 或日程开始当天的下午 2:30。
明天 16:30 / 周五 16:30相对日期或本周对应星期的具体时刻。
05-10 15:00当年 5 月 10 日 15:00。
5.10 15:00 / 5月10日 15:00更接近手写习惯的当年日期时间。
2026-05-10 14:30完整日期时间。
如果任一提醒时间晚于 DDL / 日程开始时间，或新建时已经过去，输入框会标红，保存按钮会禁用。

抽屉菜单与导航

点左上角三道杠会打开抽屉菜单。抽屉不是全屏覆盖，而是一个左侧面板，主要用于切换模块和筛选分组。

顶部区域

  • 最上方显示软件图标和 PaykiTodo 名字。
  • 底部有显示模式切换按钮。点击后会弹出 浅色、深色、跟随系统 三种模式。
  • 如果当前是 跟随系统，图标会根据系统当前明暗显示为太阳或月亮。

菜单结构

入口作用
我的任务可展开。展开后先看到 全部任务，再看到所有分组筛选项。
日历进入时间轴/周视图/月视图/列表视图所在的日历模块。
历史记录查看已完成和已取消的任务。
分组管理新建、改名、改颜色、删除自定义分组。
设置目录式设置页。大部分调试与权限项都在这里。
关于版本号、内部版本、包名、项目信息和说明入口。

分组筛选是怎么生效的

  • 在 我的任务 下点击某个分组后，首页仍然保持 已错过 / 今日待办 / 计划中 这三个板块。
  • 变化的是内容过滤：只显示当前分组内的任务。
  • 点击 全部任务 则取消过滤。

首页与待办列表

首页是 PaykiTodo 的待办主战场。它保留语录、分区、预览卡片和直接勾选完成的能力。

语录卡片

顶部显示一句本地缓存语录。右侧刷新按钮可以切到下一条，退出后会记住你停留的位置。

已错过

显示第一天及以前已过 DDL 但未完成、未取消的任务。默认展开；如果没有错过任务则自动隐藏。

今日待办

显示今天应处理的任务。默认展开。

计划中

显示未来任务，以及无 DDL 任务。默认只有在今日待办为空时才会自动展开。

任务排序规则

  • 已错过：按时间从早到晚排。
  • 今日待办：按今天内的时间从近到远排。
  • 计划中：先未来有 DDL 的任务，再无 DDL 的任务。无 DDL 会排在计划中末尾。

任务卡片怎么看

  • 左侧竖条颜色来自任务所属分组。
  • 标题前的小标签显示分组名。
  • 有备注时，卡片会直接展示备注摘要。
  • 有 DDL 时，会显示 ⏰ DDL。
  • 循环任务会额外显示 循环任务 字样。
  • 错过任务的标题、备注、DDL 会偏红，便于快速识别。

快速完成

点卡片最左侧复选框即可完成任务。完成时会有划线和烟花过渡动画，然后任务会移动到历史记录。

开始专注

长按活动待办卡片，会出现操作菜单。选择 开始专注 · X 分钟 后，会进入全屏专注倒计时；选择删除仍会先弹确认。

看板公告和桌面小组件

  • 公告现在直接写在规划台里，不再放在设置页。任意未归档规划文档中写 #公告 5.16-7.1 公告内容、- [ ] #公告 5.16 公告内容 或 > [!公告] 公告内容 即可。
  • 公告可以多条同时生效；今天在日期范围内时，每日看板顶部会显示橙色公告横幅，过期、未到期、归档或删除的公告会自动隐藏。
  • 规划台预览模式下，公告行会显示橙色“全局公告”提示；电脑端网页控制台也会在顶部显示当前生效公告，并跟随系统深色模式。
  • 公告文字较长时会在横幅里横向滚动，适合放“这段时间禁止某项娱乐”这类自我约束提醒；电脑端横幅在 60 字以内保持静止，长文本滚动时可悬停暂停。
  • Android 桌面长按空白处添加小组件时，可以选择 PaykiTodo 的 今日看板 小组件。它会展示公告、今日专注统计、今日待办、今日日程和明日日程摘要。
  • 小组件采用接近每日看板的卡片式视觉：外层叠加每日看板背景图和遮罩，顶部使用圆形菜单按钮 + “每日看板”标题 + 日期结构，不再显示额外的“轻触打开”提示。下面按每日看板顺序显示橙色公告横幅、问候卡、今日已专注卡、分组色条待办卡，以及合并今天 / 明天内容的一张日程看板卡。当前默认建议尺寸更偏 4x5 竖向看板，添加页预览也使用静态看板布局；待办行有分组色条、复选框视觉、分组标签、备注摘要和 ⏰ DDL HH:mm 文案，日程行尽量保持 App 内看板的透明行 + 竖色条节奏，进行中的日程会显示金色边框和淡金色底，弱化 RemoteViews 系统列表感。
  • 小组件支持横向和纵向拖动调整尺寸；尺寸越大，列表能露出越多看板内容。点击待办行会打开对应待办，点击日程行会打开日历详情，点击公告行会进入来源规划文档；标题和空状态行会回到默认每日看板。小组件颜色会适配系统深色模式。

专注模式

专注模式是 PaykiTodo 的番茄钟入口，用来把一条待办或一段自由专注时间变成全屏倒计时。它不会替代待办提醒，而是记录你真正坐下来推进任务的时间。

入口在哪里

  • 绑定待办：在每日看板或我的任务里长按活动待办，选择 开始专注 · X 分钟。
  • 自由专注：在每日看板的 今日已专注 卡片里点击 自由专注。
  • 每日看板仍然是阅读型看板，不额外放普通添加按钮；自由专注入口只放在专注统计卡里。

倒计时界面怎么用

  • 中间是圆环进度和大号 MM:SS 剩余时间。
  • 下方会显示 已专注 X 分钟 / 延续 N 次。
  • 暂停 / 继续 只控制倒计时，不会清空已专注时间。
  • 完成 会先确认，确认后保存为完成专注，并显示 1.5 秒完成反馈页。
  • 放弃 或返回键会先确认，确认后保存为放弃记录；放弃记录不会计入“今日已专注”完成分钟。
  • 倒计时归零后会震动，并让你选择 完成、延续 N 分钟 或 放弃。

今日专注统计

每日看板的 今日已专注 卡片只统计今天已完成的专注分钟，同时显示今天总共启动过几次专注、其中几次完成。放弃的专注会计入次数，但不会增加完成分钟。

专注设置

设置项作用
默认专注时长从待办或自由专注启动时默认使用的时长，范围 5-90 分钟。
单次延续时长倒计时归零后点延续时追加的时长，范围 1-30 分钟。
专注期间保持屏幕常亮开启后，倒计时页面会请求屏幕常亮。
专注期间压制新提醒当前版本只保存这个偏好并在专注页提示，不真实拦截系统通知。
专注记录会写入本地数据库并随 JSON 备份 / 恢复保留。数据库里记录主题、绑定待办、计划/实际时长、开始/结束时间、完成或放弃、延续次数。

创建与编辑待办

首页右下角的加号默认用于新增待办。编辑时则从任务卡片预览窗里的 修改 按钮进入。

普通待办字段

字段说明
标题必填。卡片和提醒页都会优先展示标题。
备注可为空。首页卡片最多展示摘要，完整内容在预览和提醒页查看。
所属分组决定颜色、标签和筛选归属。
设置 DDL打开后才有截止日期和提醒逻辑；关闭则变成无 DDL 任务。
农历 DDL在 DDL 卡片下方进入滚轴式农历年月日选择器，确认后转换为对应公历日期，并保留原 DDL 时分。
启用提醒只有有 DDL 的活动任务才能启用。
提醒时间使用输入框填写。可写提前分钟、当天时刻、当年日期时间或完整日期时间；输入框旁边的圆形问号会显示示例。
响铃 / 震动两者独立开关，按你的选择触发。

无 DDL 任务

  • 把 设置 DDL 关掉即可创建无截止日期任务。
  • 无 DDL 任务会排在 计划中 末尾。
  • 无 DDL 任务不会启用提醒，也不会启用循环。

循环任务开关

任务层面不额外区分“任务类型”，而是通过 循环任务 开关决定当前待办是否按循环方式生成多个实例。

目前支持的循环规则

  • 每天
  • 每周的周几，支持多选
  • 每月的第几个星期几
  • 每月的 D 日
  • 每年的 M 月 D 日

循环截止日期

  • 循环任务必须设置截止日期。
  • 默认值是首个任务日期后的 90 天。
  • 可以手动修改。

每月 D 日的特殊规则

  • 如果你选的是 29、30、31 这类日期，而某个月没有这一天，则自动落到该月最后一天。
  • 例如选择每月 31 日，到了 4 月会生成在 4 月 30 日。
  • 编辑器下方会显示一行红色小字提示这个规则。

预览循环生成

点 预览循环生成 可以先看这个循环配置大概会生成多少条实例，以及前几条的实际落点。建议你在课表、吃药、值班这种重要循环项上先预览再保存。

编辑循环任务时的范围选择

如果你修改的是仍处于活动态的循环任务，系统会先弹出“选择修改范围”。

范围含义
当前事件只改当前这一条实例。
当前事件和此后所有事件从当前实例开始，当前和未来一起改。
所有事件整个系列全部改。

待办预览、完成、取消、恢复

点击任务卡片主体会打开预览窗。很多不适合直接暴露在列表里的按钮，现在都放在预览窗里了。

活动任务预览窗

  • 显示分组、标题、备注、创建时间、DDL。
  • 底部按钮一般是 取消、关闭、修改。
  • 如果是循环任务，点 取消 或 修改 前通常会先要求选择范围。

取消任务

  • 取消后任务会停止提醒，并进入历史记录。
  • 一次性任务通常直接取消当前这条。
  • 循环任务会先要求选择作用范围。

历史任务预览窗

  • 历史列表中的卡片只显示分组、标题、备注，保持界面简洁。
  • 更多细节例如创建时间、DDL、完成/取消时间，放在预览窗里查看。
  • 历史预览窗里可以看到 恢复 和 修改。

恢复任务

历史记录中的任务可以通过 恢复 回到活动态。恢复后是否继续参与提醒，要看当前任务本身的时间条件和状态，而不是无条件重新响铃。

错过任务是怎么产生的

  • 任务过了 DDL 后，如果在短缓冲窗口内仍未完成、未取消、未改期，它会进入 已错过。
  • 进入 已错过 后仍然是活动态，不会直接丢去历史。
  • 你仍然可以点开它进行修改、完成或取消。

规划台

规划台是 PaykiTodo 的“前置思考区”。它不是替代待办和日历，而是让你先像写备忘录或 Obsidian 文档一样，把接下来要做的事、粗略时间、DDL 和拆分任务写下来，然后再识别并导入为真正会提醒你的待办或日程。

规划台当前不做 AI 自动规划、拖拽排期、甘特图或复杂项目树。它先解决最实际的问题：自由写计划 → AI 或本地规则识别 → 预览确认 → 导入待办 / 日程。

如何进入

  1. 打开左侧抽屉菜单。
  2. 点击 规划台。
  3. 如果还没有文档，系统会自动创建 我的规划。
  4. 再次进入规划台时，会默认打开你最后一次使用的文档。

规划文档怎么管理

  • 文档：打开已有规划文档列表，点击即可切换。
  • 新建：创建一篇新的规划文档。
  • 保存：保存当前 Markdown 内容。
  • 重命名：修改当前文档名称。
  • 归档：从当前文档列表中隐藏这篇文档。
  • 删除：删除前会询问确认，删除后无法恢复，建议谨慎使用。

手机端快捷栏怎么用

手机端规划台当前使用稳定的纯色 Markdown 原文编辑器和可展开的图标快捷栏。新文档默认是空白正文，示例只会作为灰色提示或说明出现，不会真实保存进文档。1.7.13 进一步修正规划台解析器，补齐行内日期日程、标题日期上下文、斜杠日期、全角分隔符和中文 AM/PM 的识别。1.7.14 增加 2 秒防抖自动保存，切换规划文档前也会先保存当前未保存内容。

手机中文输入时手打 - [ ]、缩进和标签很麻烦，所以规划台提供快捷操作。为了最大化书写区域，快捷操作默认收起，只显示 展开快捷操作 提示；需要插入任务、子任务、DDL、提醒或分组时再展开。

按钮作用
任务把当前行变成一条 - [ ] 待办；同一行重复点击不会插入多个 checkbox。
子任务在当前行下面新起一行，插入缩进后的   - [ ] 。
缩进 / 减少缩进对当前行增加或减少一级缩进。
DDL插入 #ddl 。
日程插入 #schedule 。
提醒插入 #remind 。
分组插入 #group 。
今日 / 明日插入日期分区标题；下面没写日期的时间段会按该标题理解。

长按快捷栏按钮会显示简短说明和例子。当前行是 - [ ] 大任务 时，按回车会尽量续写下一行 - [ ] ；在文档中间按回车也支持续写。

- [ ] 整理保研材料
- [ ] 写课程论文 #ddl 5.28
- [x] 已经完成的事情
  - [ ] 整理获奖证明 #ddl 5.20 #group 保研
  • - [ ] 会识别为待办候选。
  • - [x] / - [X] 默认跳过，不会重复导入。
  • 在预览状态点击 checkbox，会把原始 Markdown 行在 - [ ] 和 - [x] 之间切换；这只修改规划文档，不会直接完成已经导入的正式待办。切回编辑时会尽量定位到刚刚操作的原文行。
  • 缩进子任务第一阶段会作为独立待办导入，并在备注里记录所属大任务。

怎么写自然日程

10:00-12:30 作业1
10:00~12:30 作业1
10:00 至 12:30 作业1
明天 19:30-21:00 整理保研材料
5.28 14:30-16:00 写论文
复习 14:00-16:00
5/28 下午 2:30～下午 4:00 小组讨论
5月28日 11:55pm-次日00:30 整理材料
  • 行里有明确时间段时，默认识别为日程，不要求时间段必须在行首。
  • 如果同一行显式写了 #ddl，会优先按待办处理；例如 会议 9:00-10:00 讨论 #ddl 5.28 是一个带 DDL 的待办，不是日程。
  • 支持 -、~、～、至、到 等常见时间段分隔符。
  • 没有写日期且没有日期标题上下文时，默认今天，预览里会标出。
  • 导入日程时，默认会同时创建一条待办；这条待办的 DDL 等于日程结束时间。你可以在预览里取消。

日期上下文标题

# 明天

- [ ] 09:00-10:30 写论文
- [ ] 14:00-16:00 复习

标题可以写 # 今日计划、# 今天、# 明天、# 后天、# 周五、# 周五计划、# 5.28、# 5/28 周末计划、# 5/28周末计划、# 5月28日、# 2026-05-28。下面没有写日期的日程会继承最近的日期标题。# 收集箱 只是普通分区，会清空之前的日期上下文。

DDL 支持哪些格式

#ddl 后面可以写这些常见格式：

#ddl 5.28
#ddl 05.28
#ddl 5/28
#ddl 5-28
#ddl 5月28日
#ddl 2026-05-28
#ddl 2026/05/28
#ddl 2026.05.28
#ddl 2026年5月28日
#ddl 明天
#ddl 今天 16:30
#ddl 下午 2:30
#ddl 5.28 23:59
#ddl 5.28,23:59
#ddl 5月28日 11:55pm
  • 只写日期、不写时间，默认当天 23:59。
  • 只写时间，例如 #ddl 16:30，默认今天。也可以快速写 任务M ddl 15:00，系统会按今天 15:00 的待办处理。
  • 日期和时间之间可以用空格，也可以用中文逗号，例如 #ddl 5.28，23:59。
  • 斜杠日期和常见全角分隔符也支持，例如 #ddl 5/28、#ddl 5．28。
  • 如果解析出的 DDL 已经过期，预览会提示风险，不会偷偷改到明年。

提醒和分组

- [ ] 交材料 #ddl 5.28 #remind 5,15 #group 保研
- [ ] 下午找老师 #ddl 5月28日 18:00 #remind 2:30 pm
- [ ] 准备材料 #ddl 2026-05-30 18:00 #remind 5/30 下午 2:30
  • #group 保研 会尝试映射到现有分组。
  • #remind 5,15 表示提前 5 分钟和 15 分钟提醒。
  • #remind 16:30 表示 DDL / 日程开始当天 16:30 提醒；也支持 2:30 pm、下午 2:30、5.30 17:30、5/30 下午 2:30、5月30日 17:30。
  • #remind 5.30,17:30 或 #remind 5月30日，17:30 会按一条具体提醒时间识别。
  • 没写 #remind 时，规划台导入默认提前 5 分钟、全屏提醒、响铃 + 震动。

哪些标签暂时不当语法

#today、#tomorrow、#important、#project 目前不会写入正式待办属性，也不会自动安排日期。它们会保留在标题里，避免用户误以为已经生效。当前真正会被解析的主要是 #ddl、#remind、#group、#schedule 和 #task。

识别、预览和导入

  1. 写好规划文本。
  2. 点击 识别。
  3. 在预览中检查每条内容的类型、标题、DDL 或起止时间、提醒、分组、是否同时创建待办。
  4. 如果识别结果不符合预期，可以在预览卡片里直接修改标题、分组、备注、DDL、日程起止时间和提醒输入；DDL 和起止时间也支持 5.28 23:59、明天 16:30、下午 2:30 这类自然写法，非法提醒会标红并阻止导入。
  5. 可用 全选可导入项 / 全不选 快速控制候选；取消不想导入的条目。
  6. 没有选中任何有效候选时，导入按钮不可用。
  7. 点击 导入选中。

#imported 是什么

如果一行已经带有 #imported，规划台会默认跳过它，避免重复导入。导入成功后，规划台会自动给被导入的原文行追加 #imported，并立即保存当前规划文档。

规划台公告

公告是每日看板和桌面小组件上方反复出现的自我提醒。它不再从设置页维护，而是直接写进规划台文档，便于和阶段计划放在一起。

  • #公告 5.16-7.1 期间禁止游玩舞萌DX游戏
  • - [ ] #公告 5.16 今天必须看见的提醒
  • > #公告 5.16 引用样式提醒
  • #公告 2026-05-16 2026-05-20 本周专注保研材料
  • > [!公告] 长期提醒：先完成今天三件大事

不写日期时视为长期公告；写日期范围时按今天是否落在闭区间内决定显示。归档、删除文档或删除公告行后，公告会从看板、电脑端网页横幅和小组件里消失。尾部 #imported 或 #group 这类标签不会显示到公告正文里。

电脑端规划台

开启电脑同步后，电脑网页控制台左侧有 规划台 Tab。电脑端是左侧 textarea、右侧识别预览的简洁布局，支持文档选择、新建、删除、自动保存、识别、预览内编辑和导入选中。预览内编辑 DDL、日程开始 / 结束时间、提醒时间时，也使用和手机端一致的自然日期语法。导入成功后同样会把原文对应行标记为 #imported 并保存。电脑端支持 Ctrl+S 保存、Ctrl+Enter 识别；电脑端和手机端操作的是同一份本地数据库。生效的规划台公告会显示在电脑端网页顶部横幅，长公告会滚动且可悬停暂停，网页整体会跟随系统深色模式。

AI 识别配置

进入 设置 → AI 调用配置 后，可以填写规划台 AI 识别要用的服务名称、Base URL、API Key 和模型名。Base URL 可以填服务根地址、到 /v1、完整 /v1/chat/completions，或完整 /v1/models。编辑单个 AI 源时，可以先只填 Base URL 和 API Key，点 获取模型 从 OpenAI-compatible /models 接口拉取可用模型并下拉选择；如果服务不支持模型列表，仍可手动填写模型名。点 测试连接 会用当前模型测活，成功会显示响应字数，失败会显示 HTTP、超时或非 JSON endpoint 错误。启用后，手机端和电脑端规划台点 识别 都会优先调用已启用的 AI 源，把随手写的内容转成候选；AI 失败、超时或配置不完整时会自动回到本地规则。AI 结果也必须先进入预览确认，不会直接写入正式待办或日程；编辑、换行和自动保存不会自动调用 AI。

AI 日报 / 周报

设置 → AI 调用配置 里还有 AI 日报 / 周报 区块。你可以启用每天定时日报、每周日周报，使用 HH:mm 格式设置生成时间，也可以点击 立即生成一次日报 进行调试。准点生成依赖系统的「精确闹钟」权限；如果没有开启，PaykiTodo 会降级使用系统允许的后台调度，但可能被系统延后。

  • 日报会汇总今天完成待办、错过待办、今日日程、明日日程、明天 DDL 和今日专注分钟。
  • 周报会汇总本周完成 / 错过待办、本周日程、下周 DDL 和本周专注分钟。
  • 启用 AI 源时，报告会优先调用规划台 AI 多源轮询；AI 不可用、配置失败或网络失败时，会用本地模板生成简短报告。
  • 日报写入规划台 AI 日报 文档，周报写入 AI 周报 文档；最新报告在最上面，旧报告保留在下面。
  • 生成后会发送低优先级通知；点击通知会直接打开对应报告文档。Android 13+ 如果没有通知权限，则只写入文档、不弹通知。
  • 打开 AI 日报 或 AI 周报 时，规划台顶部会显示紫色提示条，提醒这是自动生成的复盘记录。
真实 API Key 只保存在本机设置中。备份 JSON 会保留服务名称、Base URL 和模型名，但不会导出 API Key；导入备份也不会把本机已有 Key 清空。

电脑同步

从 1.6.0 开始，PaykiTodo 内置了一个局域网网页控制台。手机和电脑在同一局域网时，电脑浏览器可以直接连接手机，查看并修改手机里正在使用的同一份待办和日程数据。

这不是单独的 Windows 客户端，而是手机端开出一个网页控制台。优点是不用再装一个桌面程序，电脑打开浏览器就能用。

入口在哪里

  1. 进入 设置 → 电脑同步。
  2. 打开 启用电脑同步 开关。
  3. 记下页面里显示的 访问密钥。
  4. 点 显示连接地址 后，记下局域网地址，例如 http://192.168.1.23:18765；关闭同步或未运行时不会显示地址。
  5. 在电脑浏览器里打开这个地址。
  6. 在网页左侧输入访问密钥，点 连接手机。

当前已经支持什么

  • 查看手机中的待办列表和日程时间轴。
  • 点击待办卡片会先打开详情预览；点击日程卡片会直接进入编辑器，删除仍需要确认。
  • 新增 / 编辑待办，支持 DDL、提醒时间、分组、循环、铃声和震动。
  • 新增 / 编辑日程，支持起止时间、全天、地点、备注、颜色、提醒方式、循环和每年同农历月日。
  • 所有改动直接写入手机本地数据库，并复用手机端现有的提醒调度逻辑。

当前限制

  • 第一版优先保证同局域网直连，不是公网云同步。
  • 电脑端仍是浏览器控制台，视觉和交互会继续向手机端靠拢，但还不是完整独立桌面客户端。
  • 模板操作、批量导入和部分更细的日历交互仍建议在手机端做。
  • 如果手机没有连 Wi‑Fi，设置页里可能看不到可访问的 IPv4 地址。

它是怎么同步到手机的

网页控制台操作的不是一份“另外的副本”，而是手机应用当前正在使用的 Room 数据库。也就是说，电脑上新增的待办和日程，会直接出现在手机端；手机端完成或删除的内容，下次网页刷新时也会同步看到。

安全性说明

  • 网页接口要求你输入 访问密钥 才能操作。
  • 如果怀疑密钥泄露，可以在设置页里点 重新生成访问密钥。
  • 建议只在你信任的局域网里开启这项功能。

遇到连不上时怎么排查

现象排查方向
电脑打不开地址先确认手机和电脑是否连的是同一个 Wi‑Fi；再确认手机设置页显示的地址有没有抄错。
网页能打开但提示未授权说明地址没问题，但访问密钥不对。重新输入，或者在手机里重置密钥。
开关打开后还是看不到地址通常是手机当前没有拿到可用的局域网 IPv4 地址，例如未连 Wi‑Fi。
手机重启后无法连接重新打开设置页确认电脑同步是否仍为开启状态；如果开启，服务会自动恢复。

分组管理

分组既是视觉颜色来源，也是筛选维度。默认分组和自定义分组都在这里管理。

你能做什么

  • 新建分组
  • 修改分组名称
  • 修改分组颜色
  • 删除自定义分组

不能直接删除的情况

  • 默认分组不能直接删。
  • 仍被活动任务使用的分组不能直接删，需要先迁移或清空相关任务。

历史记录

历史记录只保留真正结束的事项，不承接仍待你处理的错过任务。

会进入历史记录的状态不会进入历史记录的状态
已完成已错过
已取消今日待办

列表为什么这么简化

历史记录列表现在只直接展示分组名、标题、备注，是为了避免一条记录上同时塞进三个时间、两个按钮导致界面太乱。详细信息放在点击后的预览窗里查看。

全屏提醒界面

当待办或日程到点后，如果触发的是全屏提醒模式，你会看到一个独立的全屏界面，而不是仅靠通知栏轻飘飘地提醒一下。

这个界面的目标

  • 尽量在锁屏、后台、解锁后都把界面拉到前台。
  • 不允许简单按返回键就关掉提醒页。
  • 要求你明确作出一个动作，而不是继续拖延。

待办提醒会出现哪些按钮

  • 我已完成
  • 延后 5 分钟
  • 取消任务
  • 延后 X 分钟，X 来自设置里的默认延后时长
  • 自定义延后，可输入分钟数，也可输入未来具体时刻，例如 16:30；目标时间不再限制 180 分钟内。
  • 如果延后目标晚于当前 DDL，系统会同步把该待办的 DDL 改到新的目标时间。

日程提醒和待办提醒的区别

  • 日程提醒的主按钮是 我知道了。
  • 日程提醒不需要“继续延后再等你做”的那套逻辑，所以不会和待办完全一样。
  • 日程会尽量把 日期 和 时间段 直观显示出来。

延后时间的对齐规则

延后不是简单的“当前时间 + N 分钟 + 保留秒数”，而是按分钟对齐。也就是你在任意秒数点下延后后，新的提醒时刻会把秒和毫秒清零。

例如现在是 11:05:38，你点 延后 1 分钟，预期时刻应该是 11:06:00，而不是 11:06:38。

循环任务在提醒页取消时的范围选择

如果当前提醒对应的是循环任务，按 取消任务 后会先弹出范围选择：

  • 当前事件
  • 当前事件和此后所有事件
  • 所有事件

提醒权限、诊断与提示音

设置页目录现在怎么看

设置首页现在是目录式的，不再把所有内容堆成一个超长滚动页。你会看到两个大组：

  • 常用设置
  • 高级设置

提醒权限

进入 设置 → 提醒权限 后，会看到每项权限的当前状态，以及右侧的 去设置 按钮。

权限用途
通知权限允许发通知和前台服务通知。
精确闹钟让系统在更准确的时间派发提醒。
全屏提醒允许使用系统支持的全屏提醒链路。
免打扰穿透在你的系统支持且已授权时，减少提醒被静音策略压掉的概率。
忽略电池优化减少后台派发被 ROM 杀掉的风险。
辅助功能提醒作为额外兜底链路，提高回到提醒页的机会。

提醒链路诊断

  • 进入 设置 → 提醒链路诊断。
  • 点 提醒链路测试，输入多少秒后提醒。
  • 测试任务会短延迟创建，用来快速验证通知、前台服务、全屏页和无障碍兜底链路。
  • 下方最近诊断记录会显示每个阶段的状态，便于定位到底卡在哪一步。
  • 点 清空诊断 可以重置这一页的链路记录。

日历与提醒

这里主要放影响全局默认行为的选项。

设置项作用
默认延后时长影响全屏待办提醒界面里的默认延后按钮。
日历周起始日影响周视图和按周计算时，一周是从周一还是周日开始。
日历提醒默认方式新建日历日程时默认是通知栏提醒还是全屏界面提醒。

专注模式

设置 → 专注模式 可以调整默认专注时长、单次延续时长、屏幕常亮和专注期间压制提醒偏好。压制提醒当前只是偏好记录，不会真实拦截系统通知。

AI 调用配置

这里用于填写规划台 AI 识别要用的服务名称、Base URL、API Key 和模型名。Base URL 可以填服务根地址、到 /v1、完整 /v1/chat/completions，或完整 /v1/models。单个 AI 源编辑弹窗里有 获取模型 和 测试连接：获取模型会用 Base URL / API Key 调用 /models 并显示模型下拉，失败时仍可手动填写模型；测试连接用于保存前检查当前 Base URL、API Key 和模型是否能正常返回。如果服务返回 HTML 或网页内容，会提示检查 Base URL。启用后，手机端和电脑端规划台“识别”都会按列表顺序调用可用 AI 源；如果 AI 不可用，会回到本地规则识别。AI 日报 / 周报也复用这里的 AI 源，失败时会用本地模板兜底。真实 API Key 只保存在本机设置中，不会写入备份 JSON，导入备份也不会清空本机已有 Key。

提醒声音策略与提示音

  • PaykiTodo 现在把“音色来源”“播放通道”和“App 内部音量”分开配置。
  • 播放通道可以选 闹钟通道、无障碍辅助通道、通知通道 或 媒体通道。
  • PaykiTodo 内部音量 只控制软件自己播放提醒音的音量，不会新增系统级 PaykiTodo 独立音量滑块。
  • 工作模式 开启后默认不外放提醒声音，并强制使用更强震动，即使单条事项原本关闭了震动，也会避免静音无感；日程提醒也会走全屏 / 无障碍兜底链路。
  • 提醒时临时提升系统通道音量 会临时修改所选系统通道的全局音量，播放结束或提醒结束后尽量恢复原值。
  • 播放通道、周起始日和日历默认提醒方式采用下拉选择；音量百分比采用滑条与数值输入，避免按钮堆叠占用空间。
  • 设置 → 提示音 → 内置提醒音：恢复软件自带音效。
  • 设置 → 提示音 → 系统通知提示音：打开系统通知提示音列表，把它作为提醒音色来源。
临时提升系统通道音量不是 PaykiTodo 私有音量。它会短时间改变系统对应音量通道，因此默认关闭，建议只在明确需要时开启。

日历界面总览

日历模块是用来承接课程表、值班表、时间轴规划和跨天/全天安排的，不只是把待办换个壳。

顶部一排控件分别是什么

控件作用
年份月份标题显示当前中心日期所在月份。点击标题可直接调出日期选择器。
时间轴 / 周视图 / 月视图 / 列表视图模式切换按钮，点开是下拉菜单。
今天回到今天所在日期。
日历图标打开日期选择器，直接跳到某一天。

时间轴界面的总体结构

  • 最左侧固定为时间轴。
  • 上方先显示日期头。
  • 日期头下方是全天栏。
  • 最下面的大区域才是按小时划分的时间网格。
  • 红线表示当前时间；今天的日期会高亮。

四种视图

1. 时间轴

  • 最适合查看一天内的排布、持续时长和时间冲突。
  • 全天日程显示在时间网格上方的全天栏。
  • 跨天日程会按跨越的日期拆分到对应天中显示，不会只落在第一天。
  • 空白时间段点击一次会出现预添加块，再点一次才真正进入创建面板。
  • 预添加块会自动吸附到最接近的 15 分钟刻度，并默认持续 30 分钟左右的初始长度。

2. 周视图

  • 适合快速扫一周内的安排。
  • 周视图的“这一周”是受 设置 → 日历与提醒 → 日历周起始日 控制的。
  • 如果你习惯从周日开始看一周，可以在那里切换。

3. 月视图

  • 适合看某个月内有哪些安排，但不强调小时级布局。
  • 会按当前中心月份筛出该月的日程。

4. 列表

  • 按时间顺序直接列出全部日程。
  • 适合做总检查、找一条很久之后的安排，或在时间轴太密的时候快速扫一遍。

创建与编辑日程

日程编辑器比待办多了地点、颜色、全天、开始/结束时间和提醒派发方式。

基本字段

字段说明
日程标题主显示内容。时间轴卡片优先保标题，空间足够时再展示地点。

提醒设置

  • 先打开 提醒 开关。
  • 提醒时间 使用输入框填写，输入框旁边的圆形问号会显示写法说明。
  • 可写 5,15 这类提前分钟，也可写 16:30、05-10 15:00、2026-05-10 14:30 这类具体时刻。
  • 提醒方式 可选 通知栏提醒 或 全屏界面提醒。
  • 响铃 与 震动 可分开控制。

全天日程的提醒基准

全天日程没有具体开始时分时，系统默认按当天 09:00 作为提醒计算锚点。因此你设“提前 1 天”时，实际会相对这一天的 09:00 来推算。

更多按钮、周模板、学期生成

日历右上角的 … 不是杂项垃圾桶，它主要承接高效率录入功能。

批量导入

适合一次性导入课程表、值班表、活动列表。详细语法放在下一个章节单独说明。

周模板

  • 用于管理已经保存的周模板或学期模板。
  • 你可以把模板复制到某一周。
  • 也可以基于模板直接生成整学期循环日程。
  • 模板不需要你每次从零重新录课表。

保存本周模板

  • 把当前可见周内的日程保存为模板。
  • 保存时可命名，并标记模板类型，例如课程表模板、值班周模板、学期模板。

批量导入语法

如果你经常拿到一整学期课表、某周值班安排，或者想快速规划某天的时间，批量导入会比手动点格子高效很多。

入口在哪里

  • 日历页顶部点独立的 批量 按钮，可批量导入日程。
  • 我的任务页右下角点 批量待办，可批量添加待办。
  • 进入导入窗后，可直接粘贴文本，也可先点 载入示例
  • 输入框旁边的圆形问号会显示当前语法提示。

待办批量添加

我的任务页右下角有 批量待办。它是为了快速手输，所以语法比日程更轻。每日看板只负责展示今日信息，不放添加按钮。

待办批量添加的字段分隔符是英文逗号。字段内部如果需要连接日期和时间，推荐用空格或中文逗号，不要再用英文逗号，否则会被当作下一个字段。
待办格式
DDL时间,任务名称,提醒时间
示例
16:30,写报告,5
5.13 09:30,给老师发消息,09:00
5月14日，09:30,整理保研材料,5
5月14日,整理保研材料,5
无DDL,整理 Obsidian 待办
  • 16:30 这样的 DDL 表示今天 16:30；中文冒号 16：30 也会自动识别。
  • DDL 也可写 5.28、5月28日、明天、周五；只写日期时默认当天 23:59。
  • 提醒时间可写 5、16:30、2:30 pm、明天 16:30、周五 16:30、5月28日，14:30。
  • 提醒时间可省略。
  • 为了手输简单，一行待办只支持一个提醒时间。
  • 默认使用默认分组、默认响铃和默认震动设置。
  • 无DDL 的待办不能设置提醒。

最小可用语法

示例
2026-04-27: 10:20-11:55, 辅导员助理值班, @MB-B1-412; 12:30-14:00, 午休
  • 第一条必须带日期。
  • 同一天后续条目可以省略日期。
  • 条目之间可以用英文分号 ;，也可以直接换行。
  • 最少需要 时间段, 标题 这两个字段。

字段顺序

推荐顺序如下：

推荐顺序
[日期:] 时间段, 标题, 地点, 循环规则, 循环截止日期, 扩展字段

时间段写法

写法含义
10:20-11:55普通同日时间段
22:30-次日01:00跨天日程
全天全天日程

地点前面的 @ 现在怎么处理

@ 不再被当作语法标记，而是普通字符串的一部分。也就是说，@品学楼C506 会原样当作地点文字保存下来。如果你想让地点前面带 @，直接写就行。

如果内容里有逗号或分号怎么办

请用双引号包起来。

示例
2026-03-02: 19:30-21:55, "【课】二胡演奏基础", "@品学楼,C506"

写法含义

Weekly[Mon,Wed,Fri]每周一、三、五循环
MonthlyDay每月同一日
MonthlyNthWeekday每月第 N 个星期 X
Yearly每年同月同日

PaykiTodo 当前不是云同步产品，所以备份是你避免数据丢失的第一道保险。

入口

进入 设置 → 数据与备份。

你会看到哪些按钮

按钮作用
选择备份目录选择自动备份写入的位置。
导出 JSON手动导出当前所有数据。

自动备份 开关开启后，任务、模板或分组发生变动时会尝试写入备份目录。

备份里包含什么

  • 任务实例
  • 循环模板
  • 分组
  • 提醒链路诊断日志
  • 周模板 / 学期模板

导入会怎样

导入目前更偏“整库恢复”思路，而不是复杂合并。也就是说，如果你准备从一个旧 JSON 恢复，请先确认现有数据是否已经另外导出保存。

常见问题

1. 为什么我创建了无 DDL 任务，却没有提醒？

这是预期行为。无 DDL 任务只是一个不带截止时间的待办记录，不启用提醒，也不启用循环。

2. 为什么错过任务不在历史里？

因为 已错过 仍然是活动态。设计上是希望你还能处理它，而不是让它在历史里默默躺尸。

3. 为什么批量导入第一条必须带日期？

因为后续省略日期的条目会继承上一条的日期。没有第一条日期，就无法推断后面的归属日。

4. 为什么日程提醒既能选系统通知提示音，又能选播放通道？

声音文件来源和播放通道是两回事。系统通知提示音只是音色来源；真正播放时可以按设置选择闹钟、无障碍辅助、通知或媒体通道，并叠加 PaykiTodo 内部音量。

5. 为什么循环任务编辑前会多一步范围选择？

因为不先确认你是只改当前一条，还是要影响未来整个系列，系统没法安全地修改循环数据。

6. 如果我换手机，电脑现在能直接看到我手机上的待办吗？

不能直接看到。当前版本还是本地单机模式，不会自动把手机数据库同步到电脑。你需要主动导出 JSON，或者把自动备份目录放到一个可同步的位置。

7. 为什么提醒到点后最好不要只盯着通知栏？

因为真正强提醒的主链路是全屏界面、前台服务和诊断兜底链路的组合。通知栏只是其中一层，不是全部。
