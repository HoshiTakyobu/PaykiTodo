# PaykiTodo

> 一个以“强提醒”和“本地可控”为核心的 Android 待办与日历应用。  
> 面向真的需要被提醒到、需要时间轴规划、需要本地数据掌控权的使用场景。

## 项目定位

PaykiTodo 当前仍然以 **Android 本地单机应用** 为主，但已经开始补齐更完整的个人生产力链路：

- 待办、DDL、全屏提醒、锁屏提醒
- 日历、时间轴、批量导入
- 循环任务、循环日程、分组与模板
- 专注模式：待办专注 / 自由专注、全屏倒计时与今日专注统计
- 规划台：Markdown / 自然语言计划识别并导入待办与日程
- AI 日报 / 周报：自动生成复盘报告并归档到独立 AI 报告入口
- JSON 备份恢复
- 手机与电脑同局域网下的轻量控制台

它不是依赖账号体系的云平台，也不是普通轻通知型待办工具。当前更强调：

- 到点时尽量把提醒真正送到用户面前
- 兼顾后台、锁屏、解锁回前台等复杂场景
- 保持数据本地、可导出、可恢复、可观察
- 让日历与待办形成同生态体验

## 当前主要能力

### 待办系统

- 新增、编辑、删除待办
- 支持标题、备注、分组、DDL、多提醒时间
- 支持不设置 DDL 的普通待办
- 支持待办文本批量导入
- 支持完成、取消、恢复
- 支持循环任务与循环范围编辑
- 首页保留 `已错过 / 今日待办 / 计划中` 三段式逻辑
- 支持分组筛选，不破坏首页整体布局

### 日历系统

- 时间轴式日历基础已经建立
- 单日 / 多日 / 月 / 列表等视图在代码中存在并持续打磨
- 支持普通日程、全天日程、循环日程
- 支持地点、备注、颜色、提醒方式、多提醒时间
- 支持时间轴点击建日程
- 支持文本批量导入
- 支持模板和学期级生成相关能力

### 提醒能力

- `AlarmManager` 精确调度
- 通知栏提醒与全屏提醒双路径
- 待办和日程都可选择提醒方式：全屏提醒或通知栏提醒
- 锁屏、后台、解锁回提醒页等场景的多轮兜底
- 辅助功能提醒链路
- 前台服务 + 活跃提醒会话维持
- 提醒链路诊断与短延迟测试入口
- 提示音设置保留二级选择，可在内置提醒音与系统通知提示音之间切换
- 待办提醒页把“延后提醒”和“DDL 推迟”分开：延后只改下一次提醒，DDL 推迟才真正修改截止时间

### 电脑同步 / 局域网控制台

- 手机端内置局域网网页控制台
- 可在 `设置 -> 电脑同步` 中开启
- 手机与电脑同一局域网时，电脑浏览器可直接连接手机
- 当前网页端第一个入口是每日看板：显示今日日期、当前正在进行 / 下一项、今日专注统计、今日待办、今日日程和明日日程；首屏优先加载轻量看板数据，完整待办列表和日程时间轴数据按当前页面按需读取
- 当前网页端支持基础查看、创建、编辑和规划台操作
- 操作直接写入手机端同一份 Room 数据库

### 规划台

- 抽屉菜单新增 `规划台` 入口
- 支持多篇规划文档，新建、打开、重命名、归档和删除
- 默认打开最后一次使用的规划文档；没有文档时自动创建“我的规划”
- 手机端提供纯色文档式 Markdown 原文编辑器、可选 Markdown 预览、图标化横向快捷栏和多页新手教程；新文档默认为空，示例放在教程/帮助里，不再占用编辑框
- 手机端规划台快捷操作默认收起，只保留轻量展开入口；需要插入任务、DDL、提醒等结构时再展开图标工具条，优先保证书写区域
- 1.7.2 / 1.7.3 的手机端 Markdown 渲染曾因数据库迁移问题未能稳定实测；1.7.5 修复规划台数据库迁移 schema 不一致导致的覆盖安装启动闪退；1.7.8 以默认编辑、手动预览、失败保护的方式恢复 Markdown 渲染
- 本地规则解析器可识别 `- [ ]` 待办、子任务、日期上下文、常见 DDL 写法、`#remind` 混合提醒写法和轻量日程写法，例如 `明天 19:30-21:00 整理材料`
- 规划台解析器已补齐行内日期日程、时间段不在行首、斜杠日期、全角分隔符和中文 AM/PM；未实现的 `#today/#tomorrow/#important/#project` 会保留在标题里，不会假装生效
- 显式 `#ddl` 优先于自然日程识别：例如 `会议 9:00-10:00 讨论 #ddl 5.28` 会按待办处理；日期上下文标题支持 `# 周五计划` 这类紧凑写法
- 手机端规划正文会 2 秒防抖自动保存，切换规划文档前也会先保存当前未保存内容；快捷栏的“任务”会把当前行转为单个 checkbox，“子任务”会新起一行并缩进，避免同一行重复插入多个 `- [ ]`
- 规划台可识别轻量裸 `ddl` 写法，例如 `任务M ddl 15:00` 会按今天 15:00 的待办处理；仍支持标准 `#ddl` 语法
- 设置页 `AI 调用配置` 支持多个 AI 源，可新增、编辑、删除、启用/禁用并用上移/下移调整轮询优先级；启用后手机端和电脑端规划台“识别”都会优先调用 AI，把随手写的内容转成预览候选，失败或配置不完整时自动回到本地规则；AI 分组只在原文显式写出 `#group`、`分组：`、`项目：` 或 `课程：` 等标记时保留，避免从标题里误拆分组；真实 API Key 只保存在本机设置中，备份 JSON 不会导出
- 单个 AI 源编辑弹窗支持“获取模型”和“测试连接”：只填写 Base URL / API Key 后可从 OpenAI-compatible `/models` 拉取可用模型并下拉选择；仍可手动填写模型名作为兜底，Base URL 支持服务根地址、`/v1`、完整 `/chat/completions` 或 `/models`
- `AI 调用配置` 中提供 `AI 日报 / 周报`：可启用每天定时日报、每周日周报，设置 HH:mm 生成时间，也可点“了解 AI 日报”查看居中的使用指引，或点“立即生成一次日报”用于调试；报告优先调用已启用 AI 源，失败时使用本地模板；未授予精确闹钟权限时会降级为系统允许的后台调度
- 识别后进入预览，用户勾选后再导入；预览内可先修改标题、分组、备注、DDL、日程起止时间、提醒输入和日程联动待办
- 导入成功后会自动给原文对应行追加 `#imported` 并立即保存当前规划文档，避免同一行被重复导入；识别预览支持全选可导入项 / 全不选，且空选中不能导入
- 1.8.0 开始，导入后的规划行还会建立稳定映射：规划台会跟踪它对应的正式待办/日程，并在预览中显示 `已导入 / 已完成 / 已取消 / 映射丢失 / 已手动修改`
- 对于已导入但尚未完成的规划项，规划台支持“刷新已导入项”“批量顺延”“撤销上次操作”；已完成或已取消的项不会再参与刷新和顺延
- 如果正式事项和原始规划行发生偏离，规划台支持“以文档为准覆盖事项”或“以事项为准更新原文”；完成的导入任务还可以同步回 Markdown 渲染结果为 `- [x]`
- 规划台导入默认使用提前 5 分钟、全屏提醒、响铃 + 震动的强提醒策略
- 规划台支持多条公告行：在任意未归档规划文档中写 `#公告 5.16-7.1 公告内容`、`- [ ] #公告 5.16 公告内容` 或 `> [!公告] 公告内容`，也可以用手机端快捷栏「公告」按钮先插入 `#公告 ` 占位；生效公告会显示在每日看板、电脑端 Web 控制台和 Android 小组件里
- AI 日报 / 周报写入独立 `AI 报告` 归档入口（抽屉 → AI 报告），支持全部 / 日报 / 周报过滤、详情查看、删除确认和通知点击直达详情；升级时会把旧规划台 `AI 日报` / `AI 周报` 文档迁移进归档并从规划台移除
- 电脑端 Web 控制台新增 `规划台` Tab，支持同一份规划文档的编辑、自动保存、删除、识别、预览内编辑、导入、导入后 `#imported` 回写，并提供 `Ctrl+S` 保存、`Ctrl+Enter` 识别和内置使用说明弹窗；桌面端识别现在与手机端共用同一套 AI 优先 / 本地规则兜底链路，AI 返回的候选 ID 可直接导入，不再因为无法匹配本地 `line-*` ID 而出现“识别 1 条、导入 0 条”；也支持映射状态预览、刷新、顺延、撤销和冲突处理

### 看板公告与桌面小组件

- 公告从规划台正文中维护，可写多条；设置页不再提供单条公告入口
- 生效公告会在看板顶部显示橙色喇叭横幅，长公告会在横幅内横向滚动；电脑端 Web 公告横幅 60 字以内静止，超过 60 字自动滚动且悬停暂停
- Android 桌面可添加 PaykiTodo 今日看板小组件，显示公告、今日待办、今日日程和明日日程摘要；运行时和添加页预览都不再显示专注统计 / 专注一下卡片
- 小组件使用 RemoteViews `ListView`，不再固定截断为 5 条或 6 条待办，随桌面拖动尺寸自然露出更多内容；视觉上叠加每日看板背景图、顶部圆形菜单按钮、橙色公告横幅、问候卡、分区标题、圆角待办卡和聚合日程看板卡。当前小组件建议尺寸更接近 4x5 竖向每日看板，添加页预览使用静态看板式 previewLayout；顶部栏压缩为圆形菜单 + “每日看板”标题 + 日期，不再显示“轻触打开”；待办行带分组色条、复选框视觉、分组标签、备注摘要和 `⏰ DDL HH:mm` 文案，日程行保持透明行 + 竖色条节奏，正在进行的日程会有金色边框 / 淡金底，避免桌面上退化成普通系统列表；点击待办区域进入我的任务，点击日程区域进入日历，点击公告行进入来源规划文档
- 小组件颜色支持系统深色模式，并通过看板范围窄查询减少刷新时全量扫描历史数据；`1.9.12` 修复小组件查询漏取无 DDL 活动待办的问题，确保无 DDL 待办也进入小组件的今日待办区

### 专注模式

- 每日看板新增 `今日已专注` 统计卡，显示今日完成专注分钟、总专注次数和完成次数。
- 点击统计卡里的 `自由专注` 可以启动不绑定待办的全屏专注倒计时。
- 在每日看板或我的任务中长按活动待办，会出现 `开始专注 · X 分钟` 和删除入口；删除仍需要确认。
- 专注页使用全屏倒计时和圆环进度，支持暂停 / 继续、提前完成、放弃和倒计时结束后的延续。
- 倒计时归零会震动，并询问 `完成 / 延续 N 分钟 / 放弃`；完成后显示 1.5 秒完成反馈页再返回。
- `设置 -> 专注模式` 可调整默认专注时长、单次延续时长、屏幕常亮和专注期间压制提醒偏好。当前版本的压制提醒只保存偏好，不真实拦截系统通知。
- 专注记录写入本地 Room 表 `focus_sessions`，JSON 备份 / 恢复会保留专注记录和专注设置。

## 本地开发

### 环境要求

- Android Studio
- Android SDK 34
- Android Studio 自带 `jbr`

### 运行方式

1. 用 Android Studio 打开项目根目录 `Project/PaykiTodo`
2. 确认 Gradle JDK 指向 Android Studio 自带 `jbr`
3. 运行 `app` 模块

### 命令行构建

在项目根目录执行：

```powershell
./gradlew.bat assembleDebug
```
