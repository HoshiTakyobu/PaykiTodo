规划台使用说明

先像备忘录一样写成草稿；确认后发布，才进入正式待办和日程。

草稿和发布

  • 左侧“大纲事项”里新增一行，只会创建规划台草稿，不会立即出现在待办、日历、每日看板或小组件。
  • 每条草稿右侧有“发布”按钮；顶部“发布N条”会批量发布当前文档里的草稿。
  • 发布时会按节点文本解析 DDL、日程时间段和地点；发布成功后才会生成正式待办或日程。
  • 分享、拍照、语音和手机端图片识别写入的内容也先是草稿，需要人工检查后发布。

标题是用来分区的

# 收集箱：先把脑子里的事情倒出来，不会自动设置日期或 DDL。

# 今日计划 / # 今天：下面没写日期的 10:00-11:30 会按今天理解。

# 明天：下面没写日期的时间段会按明天理解。

# 本周计划：只是普通分区；重要任务仍建议写 #ddl。

常用写法

  • 待办：- [ ] 整理材料 #ddl 5.28 23:59；只写日期默认当天 23:59
  • 子任务：在任务下一行缩进两个空格再写 - [ ]
  • 日程：10:00-12:30 写课程论文、复习 14:00-16:00 或 5/28 下午 2:30～下午 4:00 小组讨论
  • 分组：#group 课程
  • 提醒：#remind 30,5 表示提前 30 分钟和 5 分钟；也支持 #remind 16:30、#remind 2:30 pm、#remind 下午 2:30、#remind 明天 16:30、#remind 周五 16:30、#remind 5月28日，14:30
  • 公告：#公告 5.16-7.1 期间禁止游玩舞萌DX游戏；公告可写多条，会显示在每日看板、桌面小组件和电脑端顶部横幅里。

日期和提醒时间怎么写

  • #ddl 支持 16:30、5.28、5/28、5月28日、明天、周五、2026-05-28 14:30。
  • 日期和时间之间可以用空格，也可以用中文逗号，例如 #ddl 5.28，23:59。
  • 常见全角分隔符会先归一化，例如 5．28、14:00～16:00。
  • #today、#tomorrow、#important、#project 暂不写入正式待办属性，会保留在标题里。
  • 提醒输入和待办 / 日程编辑界面共用同一套语法；任一提醒晚于 DDL / 日程开始时间时会阻止导入。

公告写法

公告不再从设置页维护。直接在任意未归档规划文档里写显式公告行即可，可以多条同时生效。

  • #公告 5.16-7.1 期间禁止游玩舞萌DX游戏
  • #公告 2026-05-16 2026-05-20 本周推进保研材料
  • > [!公告] 长期提醒：先完成今天三件大事

电脑端顶部公告横幅在短文本时静止显示，超过 60 字时自动横向滚动，鼠标悬停会暂停；网页整体会跟随系统深色模式。

完整范例

# 收集箱
#公告 5.16-7.1 期间禁止游玩舞萌DX游戏
- [ ] 想一下保研材料还缺什么
- [ ] 问老师课程论文格式

# 今日计划
- [ ] 10:00-11:30 写课程论文 #group 课程 #remind 5
- [ ] 晚上复习操作系统 #ddl 23:59 #remind 30,5

# 明天
- [ ] 09:00-10:30 背英语单词 #group 学习
- [ ] 完成实验报告 #ddl 21:30 #remind 15,5

# 本周计划

- [ ] 整理保研竞赛材料 #ddl 5.30 18:00 #group 保研

识别和导入

  1. 日常建议直接在“大纲事项”里一行一行写，先形成草稿，再发布。
  2. 如果需要粘贴旧 Markdown 或大段文本，展开 Markdown 兼容编辑区后再写计划。
  3. 点击“识别”，如果设置里启用了完整 AI 源，会优先走 AI；否则会回到本地规则。右侧会出现待办、日程、跳过和错误条目。
  4. 导入不会隐式触发 AI 或本地识别；如果还没有预览结果，请先点击“识别”。
  5. 在预览里修正标题、DDL、开始/结束时间、提醒、分组和备注；这些时间字段同样支持上面的自然日期写法。
  6. 可用“全选可导入项 / 全不选”快速控制候选，至少勾选一条有效候选后才能导入。
  7. 勾选需要导入的条目，点击“导入选中”；这是兼容旧 Markdown 流程的正式导入，不等同于大纲草稿发布。电脑端也支持 Ctrl+S 保存、Ctrl+Enter 识别。
  8. 导入成功后原文会追加 #imported 并自动保存，避免重复导入。
