# PaykiTodo 规划台当前设计

适用版本：v1.14.0 起。

## 1. 产品定位

规划台不是第二套待办列表，也不是语法练习场。它是正式待办和日程之前的上游备忘录：用户先把任务、课程、DDL、想法和粗略计划写下来，再由软件识别成候选项，最后人工确认导入。

核心路径：

```text
自由书写 -> 识别计划 -> 预览编辑 -> 导入选中
```

## 2. 本轮重构原则

1. 主界面只保留文档选择、使用说明、识别计划和更多操作。
2. 正文是一整块普通文本编辑区，输入体验接近备忘录。
3. 不再把井号标签、Markdown 标题、快捷语法栏、大纲节点作为当前用户教程或主入口。
4. 导入后不污染原文；已导入状态由内部映射记录。
5. 文档标题和文档日期不再提供隐藏日期上下文，避免“5 月 20 日文档里写 5.29 却导到 5.20”的错误。

## 3. 当前识别能力

### 3.1 日程

支持自然时间段：

```text
10:00-12:00 写论文 @图书馆3楼
5.28 14:00-16:00 小组讨论
明天 19:30-21:00 整理保研材料
10:00-12:00, 【课程】习思想, "@主楼B1-412"
```

地点保持用户原文，不额外补符号。日程默认不创建关联待办，除非用户在预览或编辑界面明确选择。

### 3.2 待办

支持自然 DDL：

```text
任务M ddl 15:00
交材料 ddl 5.29 23:59
明天 16:30 交材料
周五 写论文 ddl 23:59
16:30之前发邮件
```

无 DDL 普通行会作为无 DDL 待办候选，导入后统一按今日待办展示。

### 3.3 AI 优先与本地兜底

- 配置并启用 AI 源时，优先用 AI 处理自由文本。
- 明确时间段、明确 DDL 等本地高置信结果可直接使用，减少 API 浪费。
- 普通无 DDL 行属于低置信兜底，不阻止 AI 接管。
- AI 输出必须经过预览确认，不能直接入库。

## 4. 手机端 UI

- 顶部工具区：文档、说明、识别计划、更多。
- 编辑区：普通大文本输入框，展示自然写法占位示例。
- 说明：分页面解释“直接写”“常见写法”“识别导入”。
- 图片识别：识别出的文本追加到备忘录正文，并打开预览确认。

## 5. 电脑端 Web UI

- 左侧：自由文本编辑区。
- 右侧：识别预览和可编辑候选项。
- 支持保存、识别、导入、文档新建/删除等基本操作。
- 旧大纲和兼容区不再作为主入口展示。

## 6. 历史设计归档

v1.7 到 v1.13 的规划台曾尝试过标题上下文、井号标签、快捷语法栏、Markdown 预览和 Outliner。那些内容已归档到 `docs/archive/historical/PaykiTodo-1.7-1.13-PlanningDesk-Design-Legacy.md`，不再作为当前设计准则。
